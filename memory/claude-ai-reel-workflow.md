**AUDIENCE & CONTENT BAR (STANDING, set 2026-06-18):** Target a **founder / startup-operator audience that already knows Claude cold** — assume they're past the basics. **NO generic, consumer, or beginner content** (Alex flagged "write emails / summarize a PDF / receipt→budget / fridge→dinner" as "random generic video shit"). When ideating scripts, run the founder-focused research sweep and reject anything below this bar.

**Scripts = STRAIGHT VALUE, no filler (STANDING, set 2026-06-18).** Every sentence must deliver actionable value — the prompt/action to take or the concrete result you get. **CUT decorative "color-commentary" / poetic restatement lines** that just re-describe the benefit prettily — Alex flagged examples like "a wedge straight into the market they're leaving wide open" and "their careers page is a public confession of their strategy" as "random unnecessary sentences." Short functional bridges (First / Next / Then) for flow are fine; flowery closers are not. Also **don't make the narration repeat the on-screen header** (the kicker label carries the name; narration goes straight into the action).

**Scripts:** Pull the hook + framing directly from the **actual viral post** that inspired it — use its real language as the starter (e.g. "you're not supposed to prompt Claude…"). Don't write generic hooks when the source post already has a proven one.

**Hook (first 3s):** Must be uniquely interesting PER VIDEO. **Never reuse the same opening / motion-graphic hook across videos** — repetition gets boring fast. Each reel needs a fresh opener.

**Motion graphics:** High polish, but BALANCED — not cluttered, not too simplistic. Each graphic should have enough happening to be interesting (secondary motion, a couple of supporting elements, callout pops) WITHOUT becoming visual noise (Alex flagged 26-particle rings as clutter). **Every beat must be a DISTINCT *type* of motion graphic — never reuse the same graphic motif twice in one video.** Find a different visual metaphor for each line (e.g. loop=4-node cycle flowchart, subagents=branch/split, stop=clock+stacking coins, slow=prompt queue, system=horizontal pipeline — NOT the same ring/orbit reused). Vary vocabulary across videos too. Punchy cut-ins + frequent switching (Greg pacing). First 3s = fast, dense, high-energy (e.g. rapid prompt-barrage → release). **DON'T LET ONE BASE GRAPHIC SPAN MANY SCENES — even a GOOD one (Council reel, 2026-06-19, Alex: the council-TABLE "is used for way too much… I do like it at 0:15-0:16 but afterwards you use it for way too much, there needs to be way more different animations").** A single hero composition (a table, a dashboard, a chat window) reused as the backdrop for 4 consecutive beats reads as ONE long static animation even if the overlays change. RULE: pick ONE beat to be the signature shot for a recurring object and RESERVE it there; give every other beat on the same topic a genuinely different composition. **To make a graphic more interesting, ADD elements AROUND it — don't pile onto it.** Keep the clean main graphic, then surround it with **supplementary "satellite" elements in the side margins**: floating **logo chips** (GitHub, the orange Claude logo, etc.), stat pills (e.g. "🔥 946K views", "🐌 1/turn"), mini-cards (a tiny terminal). Integrate real logos wherever contextual. This is what "make it more interesting/intriguing" means — richer surroundings, not a busier center.

**HARD-LEARNED lessons from Alex's recordings (apply before processing any take):** (a) **MATCH processing to the recording — don't over-process.** Measure the noise floor first (`astats` / `volumedetect` on a gap). Aggressive `afftdn` denoise on a clean track adds a watery/artifacty quality and makes it sound WORSE — Alex flagged "audio quality not good at all" after I over-denoised.
