## HOOK RULES (verified)
1. First spoken word within 0.5s of frame 0. No greeting, no warm-up, no "in this video."
2. **Write the payoff first**, then the biggest hook that payoff can actually cash (Hoyos writes hook + last line before anything). No bait gap — sub-60% 3s-hold kills distribution (Paddy Galloway 3.3B-view study).
3. Kallaway 3-beat build (verified, 1B+ view practitioner): context clause (right viewer self-selects) → pattern interrupt → contrarian snapback. "Everyone thinks you need an agency for SEO — you don't — Claude just did 6 months of it in an afternoon."
4. Fire all three channels in second one: spoken hook + compressed 1-7 word on-screen headline (NOT a transcript of the spoken line) + striking first-frame visual. Word-one captions.
5. Design the FIRST FRAME like a thumbnail before scripting (Hoyos sketches it pre-script): most striking frame of the video, simple, ideally juxtaposition ($5k-looking deliverable + "$0" tag).
6. Lead with the after-state, withhold the how (Denney "cognitive itch"). Result stated, method locked.
7. Concrete number or power word in line one ("free," "banned," "$500/month"); vague promises banned.
8. Default to LOSS-framing: "you're wasting / stop doing / quietly killing" beats "get more from."
9. Viewer's stake, not yours: "you/your" in line one (Hormozi call-out = self-selection filter).
10. Never resolve the loop inside the hook; never ask a question the viewer can answer mentally.
11. Draft 10+ hooks, deliver 2-3 finalists across ≥2 families; model on a ≤2-week overperformer (see [[nick-saraev-style-reference]] sourcing rule).

## STORY/RETENTION ARC (30-50s, beat by beat)
- **0-3s Hook** (rules above, all 3 channels at once).
- **3-6s Foreshadow — the contract** (Hoyos, verified): one but/so sentence naming conflict + destination without revealing it. "Most people X, so [cost]. Here's the fix." Never skip.
- **6-15s Trust deposit**: pay one small instantly-usable piece by ~0:12 — buys permission to hold the main loop. (Reconcile with the GATED style: the deposit is the reframe/mechanism; the full prompts/steps stay in the DM.)
- **~15s Rehook 1**: fresh but/twist + visual switch-up; tease the last item as biggest ("the third one is the one nobody does").
- **15-35s Escalation, not enumeration**: swap-test audit — if beats 2 and 3 could swap, it's a list; rewrite so beat 3 exists because beat 2 raised the price. Every connector BUT or SO, never "and then" (Parker/Stone, verified). Rehook again ~30s.
- **35-45s Payoff**: close the main loop ONLY here, biggest number/reveal last. No resolution language ("so that's why…") mid-video — it grants permission to leave (nested loops: MrBeast memo / Hoyos final-second payoff).
- **Final 3-5s Peak-end + CTA**: CTA rides the peak, no wind-down. "Comment WORD and I'll send the exact setup" — spoken AND on screen. Hard cut ≤2s after final word. Optional loop back to line one.
- Throughout: one visual event per VO clause; something changes every 3-5s; a tight 35s beats a slow 50s.

## SOUND-LIKE-A-PERSON (the anti-buzzword layer)
1. Open sentences with And/But/So. Keep spoken transitions ("Here's the thing," "Now watch this").
2. Kill the hedge layer ("arguably," "can help to," "it's important to note"). Commit hard.
3. **BUZZWORD-KILL PASS (mandatory before delivering any script):** grep for AI-tells AND dramatic jargon — leverage, seamless, streamline, harness, robust, game-changer, unlock, empower, delve, "Mythos-tier," "load-bearing," "kill shot," "pressure-test," any phrase a friend wouldn't say at dinner — rewrite each as the plain spoken version. Read aloud once. Zero em dashes.
4. Before/afters that recalibrate: "leverage Claude's agentic capabilities to streamline your workflow" → "let Claude run the boring stuff while you sleep." "Utilize context optimization" → "give Claude one example and the answers stop sucking."
5. **⛔ NAME THE ARTIFACT (Alex, 2026-07-04: "wdym launch, that's so vague").** Abstract stake-nouns are banned: "your launch," "your work," "your content," "the thing you ship." Say the physical thing on their screen: the sales page, the pricing email, the proposal, the About page, the cold DM. Same for failure modes: not "the flaw that kills it" but "the price with no reason attached," "the three paragraphs before you say what you're selling." If a noun could belong to any video, it's too vague for this one.

## CREATOR STRUCTURE CARDS (the two voices Alex points to)
- **cindiezhu** ([[cindiezhu-style-reference]]): technical thing → **PERSONA + plain outcome** ("the diagnoser — scans your resume like a real tracker and tells you what's broken"). Show the REAL thing on screen, never describe. One complete doable loop. Strip friction out loud ("no code, about 3 minutes"). One-word comment gate.
- **raycfu** ([[nateherk-style-reference]] §raycfu): "How to [outcome] with Claude" + dollar/outcome framing ("$3k/month," "10x better answers"). Humble, conversational, zero jargon. Fresh titles scraped 2026-07-04: "Claude is dumb by default" · "How to make $8k/month YouTube channel with Claude" · "How to wipe your digital footprint with Claude" — note how PLAIN the language is: dumb, wipe, make money. That plainness IS the style.
- **⭐ raycfu VERBATIM SCRIPT FORM (3 full transcripts pulled 2026-07-04 via `python3 -m yt_dlp --extractor-args "youtube:player_client=android" --write-auto-subs --skip-download` — this bypasses the PO-token caption wall; the web client and transcript panel are gated):**
  1. Opener: "Most people don't realize…" / "Nobody's talking about this, but…" + concrete outcome ("a guy is making $22,000 a month selling a product he found for free").
  2. Third-person receipt, never "I did X": "one person ran this for 4 months, 3 hours a week" / "this developer built a file" (fits [[no-anecdote-value-first-scripts]]).
  3. Possessable NOUN + number early: "a file that makes Claude Code 10 times smarter."
  4. Body = BEFORE/AFTER blocks or numbered steps, hyper-specific: "Before: you ask for an admin check, Claude builds a 50-line permission manager class with a config system. After: it stops and asks which interpretation you meant." Specific tools/sites/quantities everywhere (USPTO bulk data portal, batches of 50, 1 in 80, $1.80/unit vs $14-22 Amazon).
  5. "This is the exact prompt I use" — POINTS at the asset on screen, never reads it out (gate-compatible).
  6. Money math stated twice (early + as the peak); friction stripped out loud ("takes 2 minutes to install," "free and public"); mid-body rehook "this is the part people miss."
  7. CTA: "I made a step-by-step guide with the exact prompts. Just comment [WORD] and I'll send it to you." + standing follow line.
  8. Voice: flat declarative, ZERO hype adjectives (no insane/crazy/game-changing). The specificity does the selling.

## SCRIPT SKELETON (fill in)
[0-3s] HOOK: number/loss claim about the viewer's money/time/results + Claude. Overlay: 1-7 word VALUE headline. Visual: the most striking frame.
[3-6s] FORESHADOW: "Most people [behavior], so [cost]. Here's [promise]."
[6-15s] BEAT 1: easiest instantly-usable piece.
[~15s] REHOOK: "But [twist] — and the last one is the one nobody does."
[15-35s] BEATS 2-3: escalating, but/so connected.
[35-45s] PAYOFF: biggest reveal/number, one-line proof.
[45-50s] CTA on the peak: "Comment [KEYWORD] and I'll send you [one noun]." Hard cut.

## PRE-SHIP CHECKLIST (all 10 must pass)
1. First spoken word inside 0.5s? 2. Number or explicit loss in the hook? 3. On-screen hook 1-7 words, not a transcript? 4. Works fully on mute? 5. But/so audit passed, zero "and then"? 6. First usable payoff by 0:15? 7. Rehook + visual switch at ~15s and ~30s? 8. Main loop open till final 8s, biggest last? 9. Hook promise fully cashed? 10. CTA spoken + on screen, hard cut ≤2s, zero buzzwords/hedges left?

**⛔ STANDING REHOOK LINE (Alex, 2026-07-05):** every future reel script gets a SPOKEN rehook at ~10 seconds in, using Alex's exact line **"But here's the part everyone misses"** (or a minimal variant), placed right before the mechanism/payoff turn. This is in ADDITION to the ~15s/~30s visual-switch rehooks. Write it into every VO draft by default.

**⛔ STANDING: NEVER SPEAK THE PROMPT (Alex, 2026-07-05):** in every future script, do NOT read the actual prompt text aloud or narrate full instructions in the VO. Instead the VO just says **"give it this prompt"** (or a minimal variant like "paste this") and the reel FLASHES the full prompt on screen (StealBox/prompt card, readable + pausable, with a "🔖 save this" affordance). The spoken line stays short; the screen carries the exact words. This keeps pacing tight, forces a save/screenshot (retention + saves), and the comment-keyword doc still delivers the copy-paste version. Applies to prompts, settings blocks, commands, and any multi-step instructions: SAY the move, SHOW the exact text.

**⛔ STANDING: GATE THE VALUE — never give away the full actionable on screen (Alex, 2026-07-06, learned from WORTHY underperforming):** WORTHY showed the complete fix in the video itself (/status typed out fully + the exact rephrase line readable) so viewers had ZERO reason to comment, and comments died. The lead magnet only farms comments if the reel creates a GAP the DM closes. Rules: (1) if the actionable is SMALL (a single command, word, or one-line prompt) — BLUR it on screen (show that it exists, not what it is) or show only a cropped fragment; (2) if it's LONG — show it long (credibility) but blur/fade the bottom half + "full prompt in the guide ↓" chip (the CLONE pattern); (3) the VO may DESCRIBE what the prompt does but never dictates it (extends never-speak-the-prompt); (4) the reel can give away the CONCEPT and the WHY for free, but the copy-paste HOW lives in the comment-gated doc; (5) before ship, ask: "could a viewer do this without commenting?" — if yes, re-gate. Balance: still show real proof/results on screen (value-first ≠ giving away the tool).

**⛔ STANDING: SIMPLEST-WORD PASS (Alex, 2026-07-06):** beyond the buzzword-kill, every script gets a plainest-vocabulary pass — for EVERY word ask "is there a simpler word a 12-year-old uses?" and swap it. Banned-by-default jargon seen creeping in: "tier" (→ level), "escalate" (→ move it up), "route/routing" (→ send/goes to), "effort setting" (→ settings), "heavy users" (→ people with big bills / the smart people), "workflow" (→ setup / the steps), "leverage/optimize/utilize" (→ use / make better), "iterate" (→ redo / keep fixing). Metaphors beat category words (worker/judge/wizard, not "model stack"). This is the cindiezhu principle enforced word-by-word.

**⛔ STANDING: AUDIENCE CALIBRATION + NAMED-SYSTEM FRAMING (Alex, 2026-07-06, from STACK v3→v4):** the audience already USES Claude — simplest WORDS yes, but never newbie-splain the concepts (v3's "it comes in three versions" got rejected as way too basic). The wanted register: insider revealing a SECRET NAMED SYSTEM. Package advice as a proprietary-sounding method "I" built ("the Pyramid Stack"), hook with name-it+proof (impossible-sounding result + the system's name + exclusivity line like "nobody's talking about this" / "Anthropic doesn't advertise this"), explain HOW IT WORKS and how to build it but keep the CRITICAL components named-but-vague (e.g. "the gate is a two-line rule", shown blurred) so the guide is required to build it. First-person system ownership ("my setup, I call it X") IS allowed and wanted for named-method reels — this refines the no-anecdote rule (still no fake stories/characters/invented receipts).
