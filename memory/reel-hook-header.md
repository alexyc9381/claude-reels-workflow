**STANDING RULE: add a `HeroHeader` to EVERY Claude reel.** Alex set this up in the "new IG reels account" session and wants it on the videos (raycfu does exactly this — a big text hook-headline pinned at the top of the reel). It makes the hook **mute-readable as a TITLE** in the first second, which is the single biggest lever on 2-second retention (the metric that drives reach — a carousel reel got 128 views / 25%-at-2s largely because it had a weak opening + no header).

**WHAT IT IS:** a big two-tone **editorial serif headline** (`fraunces` weight **900** — Fraunces has no 800; ~78–80px, lineHeight ~1.07, letterSpacing -0.02em, color INK, `textShadow 0 2px 18px rgba(236,233,226,0.96)`), positioned in the **upper-middle / natural eye-rest zone — NOT pinned to the very top** (Alex, 2026-06-22: a header at `top:~146` of a 1920 frame "is slightly too high… where my eye naturally wouldn't rest on my phone." The top ~12% is IG/TikTok account chrome — the eye rests lower.) Place the header around `top ≈ 330–360` (≈18–22% down), just above the graphic, with a clean gap; if the graphic crowds it, nudge the scene's graphic down (e.g. `translateY(40)` on the hook container) rather than cramming the header up. Reel graphic sits mid-frame, captions lower-third. It's an OVERLAY rendered ON TOP of the scenes, shown **only during the opening** (~`f2` ease-in → out ~`f92–106`, returns null after ~106 / ≈3.5s), with a line-staggered reveal (`eOut(f,2..)`, `eOut(f,9..)`, `eOut(f,16..)`). The **key keyword (`Claude`) + the hero number go in CLAY `#D2724E`** so it's unmistakably a Claude workflow and the number pops.

**⛔ THE OPENING ON-SCREEN TEXT MUST CONTAIN THE WORD "Claude" (Alex, 2026-06-23 — IG keyword SEO).** Instagram OCR-reads on-screen text and uses it for keyword discovery/ranking, so the HeroHeader (and ideally the first burned-in caption too) MUST include "Claude" so the reel gets surfaced to Claude/AI-interested viewers. The mechanism-forward headers already satisfy this ("1 Claude prompt / 1 Claude command / 1 Claude chat / 4 Claudes…") — never drop the word; if a header doesn't naturally contain it, work it in. Pairs with the clay-accent rule above (keep "Claude" + the number in CLAY).

**⛔⛔ COPY THE EXACT WINNER'S HEADER (Alex, 2026-07-10, voice memo — "make sure the header of the actual script is actually copying the exact winner header because the hook is really important… videos going forward").** When a reel is built on a comp (Door A/B), the hook-header COPIES the winner's exact on-screen header formula VERBATIM, tool-swapped only (e.g. the erase comp's frame-0 pill "ChatGPT erased my digital identity 😅" → "Fable 5 erased my digital identity 😅"). Do NOT invent a new header when the comp has a proven one — extract it from the winner-lab corpus EDL/hook frames first (the comp's header/VO channel split comes with it: the erase winner runs a CLAIM header over a QUESTION VO). Only compose a fresh header when the comp has none, and then per the rules below.

**⛔ NAME THE SPECIFIC TOOL/MODEL IN THE HOOK HEADER (Alex, 2026-07-08, "from now on"): generalizes the Claude-in-the-header rule to WHATEVER the reel is about.** The hook header must name the specific tooling wherever possible: Claude / Fable / ChatGPT / GPT-5.6 Sol / Luna / whatever product the workflow uses. Not a generic value line ("THE MOST POWERFUL AI", "GET A PROVEN WINNER") when a named-tool version is possible ("CHATGPT'S NEW SUPERPOWER", "GPT-5.6 SOL IS LIVE", "FABLE 5 PICKS THE WINNER"). Combine with the value/enticing rules below: name the tool AND convey the payoff. Reason: instant clarity on what the reel is about + IG OCR keyword SEO surfaces it to people searching that tool. Applies to every reel on the channel, Claude and non-Claude.

**⛔⛔ THE HOOK ANIMATION MUST BE A MULTI-BEAT CHARACTER STORY, NEVER ONE STATIC GRAPHIC HELD THE WHOLE HOOK (Alex, 2026-07-09, "add this to the iterative workflow to prevent it in the future").** A hook that is a single clever concept (even a mesmerizing one like a money-waterfall) held for the full ~10-12s reads as BORING / "not much going on / the same graphic for way too long" and people scroll. FIX = the hook must (1) EVOLVE through 2-3 DISTINCT visual beats mapped to the VO clauses (a new composition every ~3-4s, cross-faded), each ESCALATING, and (2) STAR THE MASCOT CHARACTERS doing things, not just abstract objects — for a Sol/OpenAI reel use SolMascot (sun) + LunaMascot (moon) + TerraMascot (earth globe), give them expressions + actions + an arc (proud→overwhelmed→relieved), so it's a little STORY. Reference fix (factory reel v4, [[factory-reel]]): the boring single Cash-Waterfall hook → a 3-beat character arc: A) Sol the shiny flagship burning money (bills spiral in + "$ BURNED" meter + the ChatGPT badge), B) Sol alone in a hardhat doing all 50 tasks by hand as a "$$$" meter races, C) Sol+Terra+Luna line up = "THE ASSEMBLY LINE". SHIP-GATE for every hook: "does the animation CHANGE at least twice, and are the mascot characters carrying it?" If it's one object doing one thing for 10s, redesign into beats. Also: if a hook uses a real LOGO (e.g. the ChatGPT knot), do NOT over-zoom/crop it — show the full mark with padding (~86% inside its circle badge, not 118%).

**⛔ DON'T GLUE PROP SHAPES ONTO A MASCOT — convey state via EXPRESSION + the surrounding SCENE (Alex, 2026-07-09, "the stuff around the sun Sol is horrible, there are just random shapes").** Sticking a hardhat + a hammer/stamp bar + a lone sweat-drop onto SolMascot read as "random shapes" and looked amateur/boring. FIX = keep the mascot CLEAN (just change its face — `shock`/`cheer`/`gaze` — to show the emotion) and let a DRAMATIC KINETIC SCENE around it carry the meaning. Reference (factory hook Beat B rebuild, [[factory-reel]] v6): "Sol expensive + overwhelmed" → NOT Sol-with-hardhat-and-hammer, but a plain wide-eyed shocked Sol being BURIED under a cascading AVALANCHE of task-cards raining down + piling up, sweat particles flying, a "$ · and SLOW" meter spinning up. Rule: if a prop isn't a clean, recognizable, well-crafted object, don't attach it to the character — use motion + expression instead.

**⛔ HOLD THE HEADER LONGER — no dead gap (Alex, 2026-06-23: "keep the header there for longer probably if the next animation hasn't come through yet").** Don't blindly fade the header at the old ~3.5s (f~106); keep it up until the opening hero's main animation has actually played / until just before the NEXT scene starts, so there's never a moment where the title's gone but the next content hasn't arrived. Tie the fade-out to the hook scene length, not a fixed 106. Content reel (9s hook → 270f): header now `if (f>252) return null` + `out = eOut(f, 236, 14)` (holds to ~8s, clears just before scene 2 at f270). Apply the same to every reel — fade the header near the end of its hook scene, not at a flat 3.5s.

**⛔ KEEP THE HEADER TITLE-BIG — never shrink it below ~74px to fit (Alex, 2026-06-23: a 66px header "reads as body text, not a title").** The header must dominate the top third like a real title. If the copy is long, **break it to 3 lines at full size** (~74–80px), do NOT drop the font to cram it into 2 lines. Content reel fix: 66px/2-line → **74px/3-line** ("1 Claude chat runs / your whole content team / for $0."). Floor ≈ 74px; standard 78–80px; 3+ lines are fine as long as the block clears the hero graphic below.

**⛔ FRONT-LOAD THE HERO — the full payoff visible by ~1–1.3s (Alex, 2026-06-23, per-second retention audit).** A slow generative/escalating hero animation must NOT leave the critical first 1–1.5s sparse — that's when ~80% decide to swipe. Audit found the content hero was near-empty at 1s (header + a lone logo, cards hadn't generated) and the impressive full fan didn't land until 3s = too late. FIX: compress the entrance so the FULL hero (e.g. all the cards) is in by ~1–1.3s, THEN let the details animate (metrics climbing, etc.) for ongoing motion. Content reel: card stagger `st = 12 + order*19, fly 16f` (full at 3.6s) → `st = 2 + order*6, fly 12f` (full by ~1.3s). Rule: by ~1s a viewer must already see the concrete payoff, not a build-in-progress. Pairs with the frame-1-concrete-hero rule.
**⛔ HARDENED — FRAME 0 MUST BE COMPLETE CONTENT, not "stuff coming in" (Alex, 2026-06-25: "the first second should have content not just stuff coming in after one second or else people scroll away").** Even a 0.3–0.5s entrance (hero scaling/fading/sliding in, $-tag slamming, header line-2 still arriving) is too slow — at frame 0 the screen looks half-empty and they swipe. So the hook's hero + header + key proof must be **fully rendered and on-screen at frame 0/1** (open ON the finished result), and ONLY then add SECONDARY motion (a slow scroll/zoom/parallax, a pulse, a counter ticking) for life. Do NOT animate the hero IN. The SITE reel still opened with a ~0.3s site+$10k+header build-in — that's the failure pattern; the fix is start at the settled state (heroIn≈1 at f=0) with just an ongoing scroll/zoom. This is now the #1 hook production rule for every reel.

**⛔ STANDING PRE-DELIVER STEP — PER-SECOND OPENING AUDIT (Alex, 2026-06-23: "add this flow in the future").** Before delivering ANY reel, extract frames at exactly 1s, 2s, 3s, 4s, 5s from the DELIVERED (7%-sped) mp4, hstack into a strip, and review each second for retention. Command: `for t in 1 2 3 4 5; do FF -ss $t -i FINAL.mp4 -frames:v 1 -vf scale=216:384 /tmp/s_$t.png -y; done` then `FF -i s_1..s_5 -filter_complex hstack=inputs=5 strip.png`. Check each second: (1) is there a CONCRETE payoff visible by 1s (not an empty stage / build-in-progress)? (2) is something CHANGING every second (no static hold)? (3) text-heavy or empty? (4) any overlap/occlusion? (5) header big + readable + holding? Fix anything that fails, re-render, re-audit. Part of the [[claude-ai-reel-workflow]] VISUAL SHIP-GATE — do it on every reel, every time, before delivering.

**ENTICING-COPY FORMULA (the version Alex approved after rejecting v1 as "not enticing enough"):** WITHHOLD the payoff — create curiosity/tension, don't flatly STATE the benefit. v1 "Claude shipped 8 hours of work" = gave it all away (rejected); the winning pattern withholds ("Claude left *something* on your desk — you haven't opened it yet" / "3 things Claude finished by 7am" — name the count, withhold the items). RULES: (a) **only TRUE numbers the reel actually shows** — fabricated metrics / scammy $ claims get killed as bait-and-switch; (b) one dominant accent, not three even lines; (c) a giant drop-cap clay NUMBER works great as the hero; (d) **spacing must CLEAR the graphic below and the captions** (Alex flagged "bottom too close to the graphic"). When unsure of the copy, run the concept pass (6 angles × 2 → judge on entice+clarity), like the overnight-reel header.

**⛔ HEADER = VIEWER VALUE, NOT THE MECHANIC (Alex, 2026-06-22 — "some hook titles aren't actually valuable, they don't communicate the VALUE to viewers immediately… 'make Claude argue with itself' makes no one actually care — the header should be the actual VALUE"):** the headline must state the PAYOFF / benefit / outcome the viewer GETS, not a clever description of what the workflow does. Describe the win, not the move. Reconciles with the withhold rule below: **state the VALUE, withhold the HOW** (the mechanic/steps are the video). Fixes applied across the whole backlog 2026-06-22:
  - "Make Claude argue with itself" → **"Make Claude catch its own mistakes"** (reliability)
  - "Make Claude ask you 20 questions first" → **"Make Claude nail it the first time"** (perfect output)
  - "Make Claude write in your buyer's words" → **"Make people buy using their own words"** (conversion)
  - "Before you use Claude, make this one file" → **"Stop re-explaining yourself to Claude"** (saves time)
  - "Stop prompting Claude / make it prompt itself" → **"Claude that works without you"** (hands-off)
  - "7 things Claude does better than agencies" → **"7 jobs you can stop paying agencies for"** (save money)
  - "One Claude command / 1,000 agents" → **"A month of work, done in an afternoon"** (speed)
  - "3 Claude prompts that feel illegal" → **"Spy on a competitor in 3 prompts"** (competitive intel)
  Test: read the header and ask "so what do *I* get?" — if it describes Claude's behavior (argue / ask / write / spawn) instead of the viewer's outcome (catch mistakes / nail it / make people buy / a month done), rewrite it. Keep it mute-legible + lead with the number/outcome.
  **AND the payoff must be INTERESTING, not just true (Alex, 2026-06-22 follow-up): "the payoff should be interesting enough — if it's not interesting enough then make the payoff sound interesting enough."** A flat/mundane benefit doesn't earn the stop; add curiosity, surprise, a vivid image, stakes, or loss-aversion. Punch-ups applied: "Claude that works without you" → **"Put Claude on autopilot"** (vivid image); "Stop re-explaining yourself to Claude" → **"Claude that never forgets your business"** (intrigue); "Make Claude nail it the first time" → **"Make Claude read your mind"** (curiosity); "Make people buy using their own words" → **"Steal the words that make people buy"** ("steal" + curiosity). Second test after the value test: "is this *interesting*, or just correct?" — if flat, reframe the same payoff with a hook device (a verb like *steal/spy*, a vivid metaphor like *autopilot/read your mind*, or loss-framing like *money you're losing*).

**⛔ NAME THE CLAUDE MECHANISM + MAKE IT SPECIFIC/INTRIGUING (Alex, 2026-06-23, on the 1000-agents reel: "this hook isn't specific or intriguing, and it's not clear it's from ONE Claude command that did this").** A value-only header ("A month of work, done in an afternoon") isn't enough — it hides HOW and that CLAUDE did it, so it's neither believable nor intriguing. The hook IS the unbelievable CONTRAST: pair the SIMPLE Claude action (one command / one prompt / one chat / one file) WITH the specific result → "One Claude command → a month of work, done before lunch." REFINES the "value not mechanic" rule: when the mechanism is itself the wow (one tiny input → huge output), NAME it. Every header must (a) name the Claude action (one prompt/command/chat/file), (b) give a specific intriguing result with a true number, (c) make the tiny-input→huge-output contrast obvious. Applies to ALL reels.

**DON'T repeat a number the graphic already shows** — complement it. E.g. carousel reel: the BEAT-A graphic already shows a big "$500" tag, so the header adds the twist instead: **"A designer's work. / No designer. / Just `Claude`."** (clay Claude) — tension + withholds the *how* (the "1 prompt" reveal is the video).

**BATCH GOTCHA (2026-06-22, added headers to the whole backlog):** before adding a header, check for an EXISTING one that may be **inline in the hook scene** (not a named `HeroHeader` component) — the Night reel's "BY THE TIME YOU'RE UP…" header was inline divs, so `grep -c HeroHeader` returned 0 and a duplicate got added (caught on the contact-sheet, removed). So grep the actual headline TEXT / top-third `top:` divs, not just the component name. Verify every reel on a hook-window contact sheet (`xstack`, not ffmpeg `tile`, to grid multiple stills) before the full render batch.

**⛔ BLUR THE *PROMPT* (the lead magnet), NOT the result (Alex, 2026-06-22 — flagged hard, twice):** the thing the reel GATES behind the CTA ("comment CAROUSEL and I'll send the prompt") must be **blurred/withheld** on screen — showing the actual prompt for free kills the reason to comment. The **result/proof (the finished carousel) stays SHARP** — that's the bait that makes people want it. ⚠️ I first got this backwards (blurred the carousel result) — Alex: "NO THE PROMPT SHOULD BE BLURRED, NOT THE CAROUSEL ITSELF." Carousel reel impl: `filter: "blur(8px)"` on the prompt-bubble text in BOTH places it appears — the hook bubble ("Make me a branded carousel") and the build-scene typed line ("Build me a 6-card carousel on this topic"); cards rendered crisp. **General rule: blur/withhold whatever you're trading for the comment (the prompt / template / list / the exact how), show the outcome sharp.** Pair with the CTA keyword in text so the blur reads as "comment to unlock," not a glitch (offer a 🔒 cue if it looks accidental).

**⛔ NEVER pre-reveal the per-scene PAYOFFS in the hook (Alex, 2026-06-23, FURIOUS: "WHY ARE YOU GIVING THEM THE EXACT COMMANDS TO USE? that is giving them the payoff so they stop watching at 7 seconds"):** if the video reveals N things one-by-one (the 5 `/commands`, the 3 prompts, the steps), the HOOK/opening must NOT show the actual names/values — that hands over the whole payoff and kills retention. **Tease the COUNT, REDACT the items.** Content-reel fix: the hook's 5 skill chips show `/` + a dark redaction bar (the command name hidden), not `/hooks /repurpose …`; each command is only revealed when its own scene types it in. Same logic as blur-the-prompt: withhold what the rest of the video exists to reveal. **BUT the redaction must READ AS HIDDEN TEXT, not an opaque blackout (Alex, 2026-06-23: "it's just blacked out but it's unclear what's behind it… it should be clear there's prompts behind there").** A single solid bar reads as a random blackout. Fix: each redacted pill = a clay `/` (the slash-command signal) + a FEW short dark redaction BLOCKS of varying widths (= redacted letters/word) + a small LOCK icon → unmistakably "a locked slash-command, name hidden," so the viewer gets that prompts/commands are behind it.

**⛔ IG FRAMING BALANCE — don't go top-heavy (Alex, 2026-06-22, on the Loops reel: "put the graphics down more, make the header slightly bigger and move it slightly down, the framing isn't good — improve the overall framing"):** the `Scene` wrapper's `translateY(-250px)` shoved every graphic into the UPPER THIRD and left the bottom half empty (top-heavy). Fix = a balanced 1080×1920 layout: **HEADER** in the eye-rest zone (`top ≈ 390`, fontSize **~76** — slightly bigger than the 68 default; bigger+lower reads better than small+high), **HERO GRAPHIC centered near frame-center** (reduced the Scene shift `-250 → -110`, graphic center ≈ y880–960), **CAPTIONS in the lower third** (`top ≈ 1250`, was 1175). Verify the hook frame ON MUTE: header top-third (below IG chrome ~top 13%), graphic dead-center, caption lower-third (above IG buttons ~bottom 18%), nothing crammed at the very top or floating over an empty bottom. When you move the graphic down, move captions down too so they still clear it (mind any bottom-anchored sub-label like a loop-counter). Likely applies to the whole backlog (shared Scene/HeroHeader pattern) — offer to propagate.
