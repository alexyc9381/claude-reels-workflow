Derived from Alex's actual performance (2026-07): the two Fable-5 reels **BLUEPRINT (31)** and **CLONE (28)** performed really well; the **META/AUTOPILOT (32)** reel underperformed. Alex told me to bake these learnings into every future script. This OVERRIDES generic advice where they conflict; apply on top of [[ig-reels-scriptwriting-principles]] and [[shortform-scripting-playbook]].

## ⛔ 2026-07-09 AMENDMENT (from the VAULT reel-38 autopsy — 5s avg watchtime)
**Clause 2 below (deadline line-1) is CORRECTED — it was misattributed as load-bearing.** VAULT followed this formula to the letter and died at 5s avg. The autopsy ([[vault-reel-premise-autopsy]]) showed the winners' real load-bearing clause was the **CONCRETE ARTIFACT/VALUE NOUN in the first breath** (CLONE: "clone its brain into cheaper models" right after the clock); the deadline only amplified it. New rules: (a) the deadline is never the stake itself and never opens two consecutive reels (lever cooldown: max 1 per rolling 3); (b) a concrete value noun lands within the first ~12 spoken words; (c) no negation openers ("the smartest move isn't…"); (d) run EVERY premise through the 9 pre-build kill-rules in [[vault-reel-premise-autopsy]] BEFORE scripting and LOG pass/fail per rule — the cerebral-payoff gate already existed when VAULT shipped past it; unlogged enforcement = no gate.

## THE WINNING FORMULA (run every Claude/Fable script against this)
The winners all share these; AUTOPILOT broke almost all of them:
1. **Subject = the hot model itself** (Fable 5), NOT an adjacent use case. AUTOPILOT was about *Facebook ads*, which filtered out everyone who doesn't run paid ads. Keep the subject the thing the whole Claude audience is already obsessed with.
2. **An expiring DEADLINE + loss frame in LINE ONE.** ⛔ CORRECTED by the 2026-07-09 amendment above — deadline is an AMPLIFIER after the concrete value noun, never the opener stake, never consecutive. (Original: "Fable 5 expires in 21 hours" etc. AUTOPILOT = no clock + no concrete noun = weak; VAULT = clock with no concrete noun = dead.)
3. **The VIEWER is the hero from word one** ("you point it at your backlog," "clone its brain"), not a company. AUTOPILOT's hero was Zuck/Meta; "you" arrived late and only for ad-runners.
4. **A possessable, NAMED artifact they keep** ("the blueprint vault", "skill files + an instructions file", "your crew"). AUTOPILOT's deliverable was a generic "setup/connector."
5. **A payoff that OUTLASTS the deadline** ("one free night pays for months", "any model starts from those files", "cheap models run your crew for pennies after"). Gives a reason the value survives the cliff.
6. **Trend-jack the exact conversation happening that week** (the Fable cutoff was THE story). Adjacent topics (Meta ads) get less save/share.
7. **High SAVE intent** = "do this TONIGHT before it's gone." Bookmarks are the #1 viral signal (verified: the overperforming Fable posts had huge bookmark:like ratios).
The proven SPINE both winners use: **hook+deadline → reframe → ~10s rehook ("here's the part everyone misses") → the one move / the named thing → outlasts-payoff → one-word gated CTA.**

## ⛔ COMMENT PLACEMENT (Alex, hard rule)
**NEVER open the VO with "comment X."** Nobody comments before they've seen value. The spoken comment CTA goes ONLY at the very END, after desire peaks. The comment mechanism LIVES IN THE CAPTION (caption opens "COMMENT [KEYWORD]" then "READ BELOW", per [[lead-magnet-docs]] convention). (CLONE opened its VO with the comment and Alex now considers that a mistake to fix, not to copy.)

**⛔ CUT THE VIDEO RIGHT ON "COMMENT [KEYWORD]" (Alex, 2026-07-08, "remember this for future videos").** End the reel the instant the CTA keyword is spoken — keep "...comment [KEYWORD]" and HARD-CUT there, dropping any trailing words ("and I'll send it to you", etc.). When splicing the VO, cut the audio right after the keyword and set CUT/durationInFrames so the video ends ~0.1s after it; never let the ending trail off. (Applied to MINT "Comment MINT" and ARENA "Comment Arena".)

## ⛔ GATE THE PAYOFF (Alex, hard rule — extends [[ig-reels-scriptwriting-principles]])
The video must make them DESPERATE for the how without ever showing it, so the ONLY way to get it is to comment. Every beat = the RESULT it produces (desire), NEVER the method. No prompts, no steps, no settings on screen. Blur/illustrate outputs (a finished page with blurred body text, a chart with blurred numbers) so they read as proof, not something copyable. Self-check before ship: "could a viewer reproduce this without commenting?" If yes, re-gate. (This is why WORTHY died — it gave the full how away.)

## GET TO VALUE IN THE FIRST BREATH
Cut all filler/yapping. Line one should hit the deadline + the directive fast ("You've got five days of free Fable 5 left. Do this..."), not throat-clear ("stop re-explaining things to it"). A tight 45s beats a padded 55s.

## ⛔ INNOVATION-BAR GATE (topic selection — Alex 2026-07-08, "way too basic, anyone can do that")
The WORKFLOW/move itself must be NON-OBVIOUS — make a normal Claude user go "I didn't know you could do that." KILL prompt-engineering-101 tricks anyone already does: "make it re-read/verify its own work" (REDFLAG, rejected), "ask you questions first", "give it 3 examples", "make it write its own prompt", "iterate N times", "add this line", "give it a persona", "think step by step". If the move is a single pasted instruction or standard prompting advice, it's DEAD. What passes = a SYSTEM: multi-agent (adversarial red-team pairs, producer/attacker/judge, expert-panel debate, parallel-variation swarm), BUILD-THE-MACHINE (make Fable build the reusable tool/agent, not do the task), SIMULATION (simulate the future / pre-mortem / role-play the market before you commit), SELF-IMPROVING loop (learns from how you correct it, compounds into a rules file), REVERSE-ENGINEER success (extract the repeatable formula behind anything great). The winners (CLONE/BLUEPRINT) were non-obvious SYSTEMS, not one-liners. Apply this BEFORE the visceral-payoff gate.

## ⛔ VISCERAL-PAYOFF GATE (topic selection — Alex 2026-07-08, do NOT repeat this mistake)
Before pitching/scripting a topic, gut-check the PAYOFF: **can the viewer instantly PICTURE it and CRAVE it?** The winners all had a payoff you can see and want in one image, a cloned brain (CLONE), build-ready specs (BLUEPRINT), 6 named employees (CREW), Fable ratting out its own lies (REDFLAG). **A payoff that's an INSIGHT / SUMMARY / ANALYSIS / "a map of your work / patterns / blind spots" is CEREBRAL and reads BORING** — Alex flagged WORLDLOAD for exactly this (clever, airtight urgency, but the payoff was fuzzy). Fix: reframe the payoff into a tangible get (money / hours / caught mistakes / a thing you keep / a named artifact), or pick a topic with a more visceral hook (fear/trust/receipts). If you can't say the payoff as a concrete thing the viewer wants, the topic is too abstract — sharpen or drop it.

## ⛔ DEADLINE-FIRST OPENER (Alex 2026-07-08, reconciles with "get to value fast")
OPEN the VO with the deadline: "You've got [N] days of free Fable 5 left..." — this is what CLONE / BLUEPRINT / CREW did and won. It is NOT "explaining what Fable 5 is" (Alex's separate complaint); it's a ONE-SECOND loss-frame that primes urgency, and the value/directive MUST land in the very next breath. Do NOT bury the deadline lower in the script (that draft got rejected). So: deadline line one → visceral value immediately after. (Update the day count to the real days-until-July-12 each time.)

## ⛔ TIGHTNESS / NO-REDUNDANT-BEATS GATE (Alex 2026-07-08, "unnecessary, people won't be retained")
Every line must add NEW information. KILL any beat that re-describes something already said, especially a "visceral result" beat that re-lists the use cases/outputs you already named (Alex flagged MINT's "out comes the finished proposal, the listing, the week of posts" as redundant, retention-killing). Top-performing reels are ruthlessly tight, momentum straight to the CTA. If the payoff is already implied by the mechanic + the use cases, do NOT add a separate result beat. Read for repetition and cut it; a tight 32-38s beats a padded 45s. Pairs with cut-dead-air in [[shortform-scripting-playbook]].

## ⛔ CLARITY / PLAIN-ENGLISH GATE (Alex 2026-07-08, recurring flag)
Every line must be instantly clear to someone who has NEVER heard the concept. Do NOT reference a mechanic (a labeling system, tiers, "the solid stuff", "the flagged lines") before you've SET IT UP plainly, and never use insider shorthand as if the viewer already follows it. Alex flagged "you trust the solid stuff and double-check the two lines it wasn't sure about" as unclear jargon. Fix pattern: spell the mechanic out in plain, visual English first ("it marks the risky lines in red, so you jump straight to those and trust the rest"). Read every line as a first-time viewer: if a phrase assumes context you didn't just give, rewrite it. This is the cindiezhu no-jargon principle ([[cindiezhu-style-reference]]) applied as a hard script gate.

## PERSONAS: SPECIFIC + NAMED + TITLED (the "exclusive team" feel)
When teaching "turn Claude into X," make each X **specific, high-value, and applicable** — NOT generic (researcher/writer/analyst = boring, Alex rejected). Give each a **NAME + a real exec title** so it feels like the viewer owns a team ("my Claude C-suite"). Reference set that landed (CREW reel): **Otto = COO** (runs the crew overnight, morning briefing), **Sol = VP of Sales** (books calls from inbound), **Nova = CSO** (competitor/price-move intel), **Remy = CMO** (a week of content from one voice note), **Vic = CTO** (builds/ships the product, catches bugs). Pick roles by desire: money, edge/intel, growth, product, autonomy.
- **Order best/most-impressive FIRST, then down the layers** (Alex overrode the classic "tease-the-best-last" for descending value).
- If using an employees/team metaphor, keep it CLEAR they're AI/Claude in different roles, not human hires (visual: one mascot clones into N tinted copies; a line like "you just set it up once"). Do NOT belabor it (Alex cut "it's still just one Fable" as clunky).
