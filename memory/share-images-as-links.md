Whenever you show Alex a sample, image, render, or any generated visual asset, **always include the clickable link/path** to the file (markdown link to the absolute or working-dir-relative path), not just an inline preview or a SendUserFile delivery. Without the link Alex often can't actually see the asset.

**Why:** Inline image previews / file deliveries don't always surface for Alex; the file path/link is what reliably lets him open it.

**How to apply:** Every time you present an image or sample, list each file as a markdown link, e.g. `[harvard-princeton-1.png](ivy-insider/out/harvard-princeton-1.png)`. Do this in addition to (not instead of) any inline preview. Applies to all asset work — see [[ivy-insider-pipeline]], [[matchtern-static-ad-pipeline]], [[social-assets-to-camera-roll]].

**⛔ VIDEOS TOO (Alex, 2026-07-09): every time a reel revision/render is delivered, END the message with the clickable LINK to the mp4** (e.g. `[38_Claude-fable5-vault.mp4](Claude-Reels-Final/38_Claude-fable5-vault.mp4)`) so he can play it immediately — "in the future if you give me a revision give me the LINK too, so I can play it here." Never just say "delivered to the folder."
