**THE CORE PRINCIPLE:** "structure gives emotion a shape." A 30-60s reel has the SAME DNA as a feature film — **setup → tension → payoff** compressed. When you cut length, keep the **emotional progression (want → obstacle → escalation → resolution), not the events.** Alex's reels keep failing because they're built as *information/a statement*, not a *problem→tension→payoff arc* — fix the structure and the hook problem mostly solves itself.

**⭐⭐ CLOSE THE LOOP AT THE END — NEVER pay off the promise mid-video (Alex, 2026-06-22, hard rule).** The hook makes a PROMISE / opens a loop. If you deliver the answer in the MIDDLE of the reel, the loop closes early and viewers drop the instant they have it — the rest of the video has no reason to be watched. RULE: the PAYOFF (the exact thing promised in the hook actually happening / being revealed) must land at the **END**. Keep the loop OPEN the whole middle; the middle ESCALATES toward the payoff (builds, raises stakes, shows the process/steps) but WITHHOLDS the final answer. Shape: **promise at the start → escalate in the middle (loop stays open) → deliver the promised payoff at the very end (loop closes) → optional loop-back line.** Reconcile with value-first ([[no-anecdote-value-first-scripts]] / the CTA gate): still give real value through the middle, but SEQUENCE it so the single biggest promised payoff is the LAST beat, never a mid-video reveal. Practical test: "If I deleted everything after the moment the promise gets answered, would the viewer have missed anything?" If no → you closed the loop too early; move the payoff to the end.

**⚠️ RECEIPTS-FIRST EXCEPTION — when NOT to withhold (added 2026-06-22, from a creator comparison vs Nate Herk / Ray Fu / Greg Isenberg).** The close-the-loop-at-the-end rule is a NARRATIVE/build-reel principle. For **capability / outcome / "receipts" reels — where the payoff is a single NUMBER or a demonstrated RESULT — the niche convention is the OPPOSITE: LEAD WITH THE PROOF on frame 1, do not withhold it.** Nate Herk demo-flexes the result up front ("it edited this entire intro you're watching"); Greg restates the quantified promise immediately; Alex's own [[claude-reel-topic-engine]] "I Gave Claude My P&L" format = "open on the number/finding, then reverse into the prompt." When you front-load the result, the open loop becomes **"HOW did it do that?"** — which carries the rest of the video. A 2026-06-22 workflow proved this live: a verifier "correctly" withheld a $14,200 figure to the end per the close-loop rule, but that moved the script AWAY from the lane — the stronger version opened on the receipt. **DECISION RULE: single-number/single-result payoff → front-load it (receipts-first). Multi-step BUILD/PROCESS or a reveal that only lands after setup (e.g. the blurred-prompt carousel) → withhold to the end.** Either way the hook still MUST be mute-readable. Also: every reel in this advanced-Claude lane needs a NICHE SIGNATURE — a holdable **architecture mental-model** (Nate Herk: "an orchestrator coordinating specialized subagents") OR a **credibility/comparison beat** (Ray Fu: "I tried Cursor, Gemini, Codex — they failed; Claude won"). See [[nateherk-style-reference]] + [[shortform-scripting-playbook]].

**FRAMEWORKS, compressed for a reel (canon, high-confidence/primary-sourced):**
- **3-Act (Syd Field):** Setup → Confrontation → Resolution = Hook → Tension → Payoff.
- **Save the Cat (Snyder):** 15 beats at FIXED PROPORTIONS → rescale to any runtime. Midpoint 50%, "All Is Lost" 68%, turn-to-finale 77%. (Page numbers are Snyder's own "approximate" guides — directionally precise.)
- **Dan Harmon Story Circle** (CliffsNotes of Hero's Journey): You → Need → Go → Search → Find → Take → Return → Change. Reel = relatable state+want (hook) → dive in → discovery + its cost → changed payoff.
- **Pixar Story Spine:** "Once upon a time… every day… until one day… because of that… until finally…" — fill-in-the-blank arc scaffold. (Function-by-function arc is solid; the exact sentence-stem wording was REFUTED — treat as approximate.)
- **Chekhov's Gun / setup-payoff:** the editing rule — everything introduced must pay off; every payoff must be set up; CUT anything that's neither. Plant the hook's promise so the ending resolves it (this enables the loop).

**SHORT-FORM CRAFT LAYER (credible but blog/NPR/Meta-level, not academic):**
- **Hook ~3s:** grabby first LINE + first IMAGE (movement/emotion/surprise). ~47% of value decided in first 3s, ~74% by 10s (Meta ad guidance).
- **In medias res / cold open:** cut the setup, start mid-action or on the boldest claim. Over-explaining up top is the #1 mistake (and was Alex's overnight-reel bug).
- **Open loop / curiosity gap:** open a question/contradiction/promise the viewer NEEDS resolved (Zeigarnik). A bold contrarian STATEMENT also works as an open loop ("never a statement" is overstated). The "blurred UNREAD brief" concept scored well precisely because it's an open loop.
- **Escalate the middle (longest ~60%):** a PUNCH every beat (twist/emotional beat/reveal), rising stakes, never flat exposition or a flat demo.
- **Payoff + loop ending:** resolve the loop the hook opened, tie the last line back to the first for rewatches.

**MAKE VALUE/AI CONTENT FEEL LIKE A STORY (Alex's core need — no obvious narrative):**
1. **Micro-arc (highest value):** wrap any how-to/list/demo as **problem → action → resolution** (NPR: hiccups→vinegar→cured). Steps become a story once bracketed by a problem and a resolution.
2. **Make the VIEWER the protagonist** — no character needed. Their problem = Act 1, your method = the journey, their new ability = the transformation.
3. **Give Claude/the product a narrative ROLE not screen time** (3 models, single-blog taxonomy, use as working tool): **Reluctant Hero** (appears only after other approaches fail), **Constant Companion** (present throughout, never pitched), **Reveal Object** (the earned payoff — only if Act II built curiosity).

**⭐ REUSABLE SCRIPTING CHECKLIST (run every script through it):** (1) HOOK 0-3s: grabby line+visual, open a loop, cut all background; (2) PROMISE/STAKES: what they get or what's at risk; (3) ESCALATE (long middle): a punch each beat, rising stakes, no flat demo; (4) PAYOFF: resolve the EXACT loop the hook opened (Chekhov); (5) LOOP + one-line CTA; (6) throughout: preserve the emotional progression, cut anything that isn't a setup or payoff; (7) SPECIFICITY ([[specificity-effect]]): every claim anchored by an exact figure, a time dimension, or a named term-brand — a vague claim reads as selling; (8) climb the [[dopamine-ladder]] (L1 stun-gun → L2 loop → L3 anticipation → L4 NON-OBVIOUS payoff) and hold the loop to the end.

**BEAT TEMPLATE (scale to length; shown @40s):** 0-3s hook/open-loop · 3-7s promise+stakes · 7-28s escalating middle (3-4 punches) · 28-36s payoff (loop resolves) · 36-40s loop-back + CTA. (A 60s blog scheme: Hook 0-8s / Tension 8-45s / Payoff 45-60s — one reasonable scheme, not a law.)
