# The Specificity Effect

Principle (creator's verbatim teach, via Alex): "When you're explaining something, be as specific as possible — numbers, percentages, names, and as many quantifiable details as you can." **Why it matters:** a GENERALIZED statement is clocked by the viewer's brain as **selling**, because there's no provable anchor to wrap the claim around. A SPECIFIC statement signals instant trust — it *feels real*.

- ❌ "I changed one thing and it got me way more views" → reads like BS; "way more views" has no anchor.
- ✅ "I changed one word in my thumbnail and it took my views from 1,700 to 1.2M" → extremely specific, so it feels real and pulls interest.

## The 3 levers (apply at least one to every claim)

1. **Exact figures that qualify a stated result** — `1,700 → 1.2M`, `52% → 88%`, `$3k/mo`, `0/8 → 8/8`. A number beats an adjective.
2. **A time dimension** — "in just 2 weeks" beats "fast"; "overnight", "in one sitting", "before a human ever reads it". A concrete duration is a specificity anchor.
3. **Name / term-brand the concept** — coin a proper-noun name ("The Specificity Effect", "The Dopamine Ladder", "The Judgment Vault"). Even a made-up name feels *more real* because it sounds specific. This is also the keyword/artifact-naming lever we already run (name the ARTIFACT, name the SERIES).

## How to apply (Script Factory)

- **Stage 3 (draft) — SPECIFICITY PASS:** enumerate EVERY value/result claim in the draft; each must carry an exact figure, a time dimension, or a named/term-branded concept. Any generalized claim with no provable anchor ("way more", "a lot faster", "tons of X") gets rewritten to a specific one BEFORE it leaves the draft. Numbers may be honest demo/estimate figures (label them), never fabricated receipts — see the TRUTH-REQUIRED rule.
- **Stage 4 (gate):** RULES LOGGER owns the SPECIFICITY CHECK (scored in **concreteness**): flag each unanchored value claim + supply the specific rewrite; an unanchored PRIMARY claim caps concreteness ≤6 so it can't ship until fixed. COLD VIEWER corroborates in **believability** (a value claim with no exact figure/time/name reads as SELLING = scam-smell). Honesty guard: a specific number that isn't real is worse than a vague one — specificity must ride on a TRUTH-REQUIRED-clean claim.
