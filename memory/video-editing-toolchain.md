Set up 2026-06-15 for Matchtern video edits. No system ffmpeg/cmake/whisper; this is the path that works on this machine (Apple Silicon, node 24 at `~/.local/bin`):

- **ffmpeg/ffprobe:** npm `ffmpeg-static` + `ffprobe-static` (no brew). Installed under `~/Downloads/matchtern-longform/tools/`.
- **Transcription:** `whisper.cpp` built from source at `~/Downloads/matchtern-longform/tools/whisper.cpp`. cmake was missing → install via `pip install --user cmake` (gives a 4.x binary). Build with `SDKROOT=$(xcrun --show-sdk-path)`. Model `ggml-small.en.bin`. Word-level timing: `whisper-cli -ml 1 -sow -oj`. ALWAYS hand-correct Whisper's transcript before burning captions (it mishears brand/jargon words).
- **Editor:** Remotion 4.x (`video/` subdir), fonts via `@remotion/google-fonts`. Render 1920×1080 h264. Transcode 4K→1080p H.264 first (headless Chrome decodes 4K/HEVC unreliably). Verify with `remotion still --frame=N` before the full `remotion render`.

Used by [[matchtern-longform-video-style]] and [[matchtern-design-system]].


**⛔ "CUT CUT" single-take retake removal (Alex, 2026-06-23 — records straight-through and says "cut cut" to kill a flubbed take):** when a VO has verbal "cut cut" markers, splice out the flubbed take + the marker, keep the clean retake. METHOD: (1) the word-level whisper (`-ml 1 -sow -oj`) often HIDES the flub — it stretches/merges the bad words and may drop the retake's first word, so DON'T trust word times here. (2) Find the real boundaries empirically: transcribe short ISOLATED segments (`ffmpeg -ss A -t N` → whisper `-nt`) around the tail to locate "cut cut" vs the clean retake, AND run a fine energy scan (`volumedetect` every 0.1s) to find the actual speech/silence dB boundaries (silence ≈ below ~-30dB). (3) Make ONE clean cut FROM THE ORIGINAL wav (not compounded splices — those drift): `aselect='not(between(t,CUT_START,CUT_END))',asetpts=N/SR/TB`, cutting from the silence after the last good word to the silence just before the clean retake (leave ~0.05–0.3s natural gap). (4) Trim leading/trailing silence (`silenceremove…areverse…`). (5) Re-align captions with the canonical-BEATS bucket method ([[caption-sync-gate]]), recompute the `L` array + `durationInFrames`. Built reel 11 (content-team / 5 Claude content skills) this way.

**⛔ NEVER use the 16 kHz whisper wav as the reel's actual VO (reel 15 SEO, Alex 2026-06-26: "the audio quality is quite bad — is there a reason?").** The transcribe step converts the m4a to **16 kHz mono** for whisper; that file is telephone quality (band-limited to 8 kHz) and sounds muffled/thin as audio. I accidentally shipped the 16 kHz `vo_seo.wav` as the `<Audio src>`. FIX: keep TWO wavs — 16 kHz for whisper only, and a **full-quality 48 kHz** one for the reel (reproduce the SAME splice/trim chain from the original m4a at `-ar 48000`). Pre-deliver check: `ffprobe public/vo_*.wav` → `sample_rate=16000` is the bug. Helpful VO polish chain: `highpass=f=75, acompressor=threshold=-19dB:ratio=2.5, highshelf=f=7500:g=2.5, loudnorm=I=-15:TP=-1.5`.

**⚠️ Alex's VO recordings land in ~/Downloads as `Untitled - M:DD:YY, H.MM PM.m4a` — reference via GLOB, not the literal path.** The displayed time ("10.20 PM") contains a NARROW NO-BREAK SPACE (U+202F) before AM/PM, so a hand-typed path fails ("No such file or directory") even though it looks identical. ALWAYS copy via a `*` glob first: `cp ~/Downloads/Untitled\ -\ 6:23:26,*10.20*PM.m4a /tmp/vo_src.m4a` (the `*` covers the weird space). Then whisper with `word_timestamps=True`. Workflow for a recorded reel: transcribe → identify which script → trim lead/trail silence (`-ss`/`-to`) → build `src/data/words_<reel>.json` ({word,start,end,line}, shift word times by the lead-trim, subtract ~0.08s lead per [[caption-sync-gate]]) → derive `L[]` scene array from each line's first onset → model a new `Claude<X>Reel.tsx` on `ClaudeContentReel.tsx` (the cleanest template: palette, ClaudeMark/Bloom/Sheen/Scene/Captions/Sfx, music bed ≤0.2) → register in Root → render → contact-sheet ship-gate → 7% speedup → deliver. Built reel 12 (Claude $10k-websites, SITE) this way 2026-06-23. Reel 13 (CONTROL / "Claude can now control your computer") the same way 2026-06-25 — note its VO had TWO "cut cut" retakes (whisper hid the 2nd until after the 1st splice; ALWAYS re-transcribe the spliced wav and re-check the `cut` count until 0).

**⛔ "I don't hear the riser / sound effects" is almost always GAIN-MASKING, not missing files (Alex, reel 16 v2 2026-06-27).** Before regenerating assets, ffprobe + `volumedetect` every referenced sfx. The bug pattern: the `<Sfx>` per-clip `v=` multipliers were tuned too low (riser at v=0.36 on a -17dB file → effective ~-26dB) while the VO plays at full 1.0 (-19dB mean / -1.5dB peak) and a riser is a wideband sweep occupying the voice band → perceptually masked the instant VO+music start. FIX: (1) use a louder/longer opening asset — `metal_riser.wav` (1.95s, 0dB peak) at v≈0.85 beats `riser.wav` (0.75s) at v=0.36; (2) push the opening one-shots HOT since they're brief — sub→0.8, boom→0.7, whoosh/shimmer/snap→0.5-0.6 (low-freq sub can go loud safely, won't fight the VO); (3) start the music-bed fade-in slightly later (~1.6s) so the riser is clear under the hook; quiet generic one-shots (ding/data/sparkle/angelic) need v≈0.45-0.55 to clear -38dB effective. Verify: `ffmpeg -ss 0 -t 2 -i out.mp4 -af volumedetect` opening max should hit ~-1 to -4dB, not ~-20dB.

**⚠️ `ffmpeg` EATS STDIN inside a `while read` loop (2026-06-27) — pass `-nostdin` (or `</dev/null`).** Downloading+converting a list in `while IFS='|' read name url; do curl ...; ffmpeg -i ... ${name}.jpg; done < list.txt` silently corrupted every filename after the first (ffmpeg consumed the leading char of each next line — `gx_rezo`→`x_rezo`). ALWAYS use `ffmpeg -nostdin ...` in read-loops. Also: Behance full-res images are at `mir-s3-cdn-cf.behance.net/project_modules/(1400|max_1200)/...`; galleries lazy-load so static HTML often exposes only the og:image/first module — fetch several galleries (one hero each) per query rather than expecting many from one page.

**⚠️ Playwright MCP screenshots are SANDBOXED — not reachable from the Bash tool (2026-06-25).** `mcp__plugin_playwright_playwright__browser_take_screenshot` saves to "./<name>.png" in the Playwright server's own (containerized) filesystem; `find`/`mdfind`/`~/.playwright-mcp` all come up empty on the host. So you CANNOT grab a real-website screenshot that way to use in a Remotion `<Img>`. Options for "a real site in the reel": reuse existing `public/refs/bh_site_*.jpg` (cropped Behance), or have Alex screenshot the site himself. **Use a COMPLETE/tall full-page crop (`bh_site_a_tall.jpg`) shown at full browser width + scrolling — never a wide hero cover-cropped into a narrow frame** (that's what made the LUXE crop look "cut off" — Alex flagged it).

**⭐ AGENT/"Claude-controls-your-computer" reel engagement pattern (reel 13, Alex wanted the hook "more interesting, more stuff going on"):** build a persistent Mac BROWSER (`Browser` w/ chrome + a "Claude · working" pill) that an animated CURSOR drives — `Cursor` (mac-arrow SVG + a click-ripple), a `path()` interpolating waypoints, `Target` glow-boxes on what it's about to click (flash on click), and a live **ActivityFeed** (a dark glassy panel, lower-right, streaming action chips: ✓ done rows + a ● spinner "working" row appearing every ~0.7s) = the "lots going on".
