**⭐ REAL-SCREEN CAPTURE — headless Chrome to disk (works on this Mac):** system Chrome exists at `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`. Screenshot ANY live URL straight to a file (no MCP/sandbox issue, unlike [[video-editing-toolchain]]'s Playwright note):
`"$CHROME" --headless=new --disable-gpu --hide-scrollbars --force-device-scale-factor=2 --window-size=1280,1900 --virtual-time-budget=9000 --screenshot=/tmp/out.png "https://github.com/..."`
→ a real 2560×3800 retina PNG. `--virtual-time-budget` waits for JS/render.

**⭐ WHEN A SITE BOT-BLOCKS HEADLESS CHROME (hangs/exits 1, e.g. z.ai) — use thum.io (server-side screenshot, free, no key):** `curl -sL --max-time 60 "https://image.thum.io/get/width/1280/fullpage/noanimate/<url>" -o page.png` → real screenshot rendered on their server (a real bot, not yours). `/fullpage` = entire scrolling page (can be very tall, e.g. 1280×6566); drop it for above-the-fold; `/noanimate` freezes intros. **⭐ NICK-SARAEV ANNOTATION SYSTEM (in `ClaudeBrainReel.tsx`, reusable):**
- `Browser{w,h,url}`: mac chrome (3 dots + URL bar with 🔒) wrapping a screen. KEEP IT SHORT (~880px tall, top ~600) so it ends ABOVE the caption zone — captions are dark INK on cream and are ILLEGIBLE over the dark GitHub screens, so never let the browser run under top:1500.
- `Screen{img,vw,vh,focus,scale}`: pan/zoom over a tall screenshot. `scale`≥1 zooms; `focus` (0-1 frac of image height) sets the vertical center. Animate focus+scale across the scene with `eio()` for the slow Saraev push-in. Hardcode each shot's aspect in `SHOT{}`.
- `HiBox{x,y,w,h,at,color,label,lblSide}`: highlight box that DRAWS IN (height ramps) with a glow + an optional dark label chip. Point it at the thing the VO names.
- `Callout`, `Cursor{x,y,click}` (mac arrow + click ripple, move along an interpolated path), `Term{}` (dark terminal that TYPES a query char-by-char then reveals an answer citing "your notes" — for an "ask Claude" beat).
- Frame-0 rule still applies: scene-0 = the real screen + hook-header FULL at f0 (a real screen on frame 0 is itself the top retention fix, see [[design-reel-real-samples]] frame-0 note + the "looks-like-an-ad" teardown in [[reel-retention-hook-teardown]]).
