**Why:** abstract placeholder geometry (slate bars/blocks) reads cheap; a real, polished design screenshot instantly elevates the scene and looks premium. Ties to the [[claude-ai-reel-workflow]] (13) VISUAL-FIRST rule and the real-asset editorial style.

**Copyright (resolved — don't re-litigate every time):** Behance projects are individual designers' copyrighted portfolio work and Adobe's terms don't grant reuse rights, so embedding one in a published/monetized reel is a real infringement risk. I flagged this and offered safe alternatives (his own Matchtern site, royalty-free Unsplash, a realistic generated mockup); **Alex chose Behance anyway and accepted the risk, treating the art as reference/placeholder.** So: just do it going forward — don't re-ask each time. **How to apply (working method, verified on the $297 offer reel):**
1. `WebSearch` behance.net for the design type (e.g. "behance.net landing page website design"); grab a live `/gallery/<id>/...` URL.
2. `WebFetch` that gallery page asking for the og:image + any `mir-s3-cdn-cf.behance.net` image URLs (the page's own HTML 404s/JS-renders sometimes — try a couple galleries).
3. `curl` the CDN image down with a browser UA + `-e https://www.behance.net/` referer (hotlink works): e.g. `mir-s3-cdn-cf.behance.net/project_modules/1400_webp/<hash>.png`.
4. **Behance `project_modules` images are tall PRESENTATION DECKS** (e.g. 1400×11932), NOT clean screenshots — they have the designer's name, "design concept" labels, laptop mockups, etc. SCAN the deck for a FLAT full-bleed website screen: `ffmpeg` crop horizontal bands at stepped y-offsets → `vstack` montage → eyeball which band is the clean website hero, then crop that exact region (skip the presentation labels above it).
5. Composite into a **browser-window frame** (Mac chrome: 3 traffic-light dots + a URL pill) via `<Img src={staticFile("refs/<file>.jpg")} objectFit:"cover" objectPosition:"center top"/>` filling the area below the chrome. Keep the Sheen/Glint on top.
6. Save assets under `public/refs/`. Reference impl: `ClaudeOfferReel.tsx` Example scene (`refs/site_offer.jpg` = a real Behance SaaS/agency landing hero inside the browser). `Img` + `staticFile` already imported from remotion.

Can extend the same real screenshot to other "your offer" beats (the torn-document Foreshadow, the under-attack Objections card) for cohesion if asked.

**⚡ FASTER scrape method (verified on reel 12, the SITE reel) — curl the gallery SSR, don't browser-scroll:** `curl -s -A "<chrome UA>" -e "https://www.behance.net/" "https://www.behance.net/gallery/<ID>/x" | grep -oE 'project_modules/[a-z0-9_]+/[a-f0-9]+\.[a-f0-9]+\.(png|jpg)' | sed -E 's#project_modules/[a-z0-9_]+/#project_modules/1400/#' | sort -u` → gives the first ~3-6 module URLs (SSR-exposed) WITHOUT needing the JS-rendered page. Then hotlink-curl each `mir-s3-cdn-cf.behance.net/<path>`. To get the gallery LIST + IDs first: Claude-in-Chrome to `behance.net/search/projects?search=<q>`, scroll, scrape `a[href*="/gallery/"]`. **KEY: the TALL modules (height > ~3000px, check with ffprobe) are FULL-PAGE FLAT website screenshots** — one long scroll of the whole site. Slice ONE into multiple distinct sections (hero / stats / pricing / FAQ) with `ffmpeg crop=1400:900:0:<y>` → several clean "different premium sites" from a single download (browser-viewport aspect ≈ 1.56, so crop 1400×900). Composite each into the Mac browser chrome (`BrowserBase`/`SiteHero` in `ClaudeSiteReel.tsx`). For a MOBILE/phone view, DON'T scale a whole desktop screenshot into a phone (squishes — Alex flagged "site looks horrible on phone") — put the desktop `<Img>` at `height:100% width:auto left:50% translateX(-50%)` so it COVER-crops the centered hero = a real mobile look (`PhoneSite`). Caveat: most search-result decks are messy (laptop/phone mockups, "Thanks for watching" slides, text concept slides) — only the tall full-page modules are clean; SSR curl on some galleries returns nothing (then browser-scroll). Reel 12 used `infer_2` (one 1400×8260 full page → 3 sections: hero/27M-stats/pricing) + `polyai_1` (dark site) = 4 distinct screens.
