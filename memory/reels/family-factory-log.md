### The mechanism, in one sentence
**Genealogy's hard problem is record LINKAGE, not record retrieval.** Proving the Vincent Kowalski in the
index is *your* Vincent Kowalski needs corroboration across the exact indexes walled to Claude — so Claude
retrieves the name and **generates the kinship**. It does not hallucinate the person; **it hallucinates the
relationship, and the relationship is the entire product.**

### The receipts (re-runnable, observed not predicted)
- **Every corroborating index is unreachable:** FamilySearch = sign-in wall (Claude cannot create an account
  to pass it) · Ancestry = paywall · 1950census.archives.gov = JS-only shell, no public name-search API ·
  FindAGrave = HTTP 403 · Chronicling America = 403 on both hosts.
- **Constraint-relaxation-until-match is the DEFAULT, unprompted:** one query (Vincent Kowalski, Chicago,
  d.1935) returned **four different same-named strangers across four states, 1925–1951**, with the model
  volunteering *"the year might be slightly different."* That is the amateur-genealogy error that
  manufactures wrong trees — reached by default, not under pressure.
