Spin-to-win email-capture site branded **Alex AI** at `~/Downloads/claude-max-spin`. Viewers type a code + email and spin a wheel for a **free Claude Max subscription**; every email is captured to a mailing list. Built + iterated 2026-07-07. **⭐ LIVE at https://spinthevault.com** (deployed 2026-07-07 on Vercel project `claude-max-spin`, scope `alexyc9381-2018s-projects`, DB = Turso `spinthevault`; GoDaddy nameservers point to Vercel). Admin/email export at `/admin?key=<ADMIN_KEY>` (key handed to Alex, not stored here). Env vars (TURSO_*, ADMIN_KEY, ACTIVE_CODES=the 14 guide codes, WINS_ENABLED, etc.) set at the Vercel project level — change a knob = edit the Vercel env var + redeploy (`vercel --prod`), or ask. The 14 guide codes are printed in the Claude Reels lead-magnet .docx files (see ~/Downloads/claude-reels-spin-codes.md for the guide→code map).

Architecture: Express app in `app.js` (exported, no listen) → `server.js` runs it locally (`npm start`, http://localhost:3000), `api/index.js` runs it as a Vercel serverless function (`vercel.json` rewrites all routes to it + `includeFiles: public/**`). DB is **libSQL/Turso** via `@libsql/client` in `lib/db.js` (ASYNC): local dev/tests use a SQLite file (`DB_PATH`, default `./giveaway.db`); production uses Turso (`TURSO_DATABASE_URL` + `TURSO_AUTH_TOKEN`) and auto-picks the pure-JS `@libsql/client/web` (no native module to bundle). Migrated off better-sqlite3 2026-07-07 so data persists on serverless. `Dockerfile`+`fly.toml` also present as a container-host fallback. Tests are async (6, all pass); tests use a temp FILE db (libSQL opens txns on a separate connection, so `:memory:` isn't shared).

Earn/spin model (NO login, server-side): **1 spin per code** + **1 spin per invite**. `ACTIVE_CODES` is a COMMA-SEPARATED list (multi-code, one per guide doc + secret codes hidden in videos) — server matches the entered code against the list (case/space-insensitive) and records which one; each distinct code = one spin per person. Referral: after a first spin each player gets `origin/?ref=<token>` (localStorage); a friend spinning via it credits the referrer +1 (guards: once per invited email, no self-referral, cap `MAX_INVITES_PER_PLAYER`=100, only on a real spin). Balance = redemptions + invites_earned − spins_used. Winner cap + `WIN_PROBABILITY` (1/100000) + `WINS_ENABLED` kill switch in `config.js`. Emails: `/admin?key=ADMIN_KEY` → CSV (email, joined, codes_used, won, spins_used, invites_earned) + `/admin/emails.txt` + `npm run export`; auto-lowercased/deduped.

Anti-abuse (added 2026-07-07, live): `lib/validation.js` `looksFake()` blocks obvious junk/test emails BEFORE any DB write (reason `'fake'`) — exact set (test/user/demo/asdf/qwerty…), fakeword+digits like `user6`/`test123`, bounded test/fake/dummy/placeholder tokens (`flowtest`), and 4+ repeated chars (`aaaa`); real names like `john.smith`/`greatest`/`a.patel22` pass. Layered with: disposable-domain blocklist + MX/DNS check, an in-memory per-IP rate limiter (`IP_SPINS_PER_HOUR`=12), and a **per-IP account cap** — `config.maxEmailsPerIp` (`MAX_EMAILS_PER_IP`, default **20**) new accounts per IP in a **rolling 24h window** (`created_ip` column on players; window keeps it CGNAT/mobile-friendly so it never permanently locks a shared IP). **Tuning knob: if real mobile viewers report being blocked, RAISE `MAX_EMAILS_PER_IP` in the Vercel env + redeploy.** Verified live: `user6@gmail.com`/`flowtest@gmail.com` → `fake`; real email + real code works. 10/10 tests pass.

Design (Alex's rules, 2026-07-07): light cream Claude theme, NO gradients; flat wheel with ICONS (🎁 prize on the lone clay slice, all duds 💨); title "Alex AI" + a GENERIC spark mark (NOT Anthropic's real logo — trademark caution; keep "Claude" out of the DOMAIN too); prize stated directly ("free Claude Max subscription"); big wheel (great on mobile); **spotlight spin** (taps pop the wheel into a centered enlarged dark stage, quintic slow-crawl before it lands); synthesized audio ON by default (ambient music loop that ducks on spin + whoosh/ticks/riser + crowd "ooooh…oh!" GROAN on loss + fanfare/applause + confetti); tap the WHEEL to spin (empty fields → "do steps 1 & 2 first" nudge); NO odds or global counters shown (show own spins-left + invite count); removed "No access, no prize."; em-dash-free copy per [[no-em-dashes-in-copy]].
