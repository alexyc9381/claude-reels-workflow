## The pipeline — run EVERY reel, every revision
1. **Script/Hook self-gate** — clear the winning formula + HOOK GATE before building (see [[reel-winning-formula]], [[claude-reel-hook-library]], [[ig-reels-scriptwriting-principles]]).
1b. **⛔ HOOK IDEATION** (Alex 2026-07-08 addition) — before building, brainstorm **5 distinct opening-hook / first-frame visual concepts** (the memorable opening scene, like CLONE's rocket or BLUEPRINT's wizard falling from the sky), **RANK them best-to-worst** against the hook criteria (mute-readable as a title, stops the scroll in the IG safe zone, on-topic to the payoff, novel not an ad-card), and **pick the best one to build**. Show the ranked shortlist reasoning, then commit to #1.
2. **Build** — characters are the **costumed Claude pixel sprites** (clay Mascot + scene outfits: hats/suits/helmets/masks/armor/capes, like the CREW crew). Never a bare/placeholder shape.
3. **Render + contact sheet** (montage key frames incl. the LATE part of each scene — that's where dead spots hide).
4. **⛔ THE ITERATION LOOPS — run a FEW passes, not one** (Alex 2026-07-08). After the first render, do NOT deliver. Loop through the reel MULTIPLE times; each pass raises the bar until nothing is boring and retention is maxed. Every loop hunt for + fix:
   - make the animation **much more interesting** (more detail/layers/motion; thin scenes get more);
   - **animation color palette** (warm cel-shaded, NEVER neon, no dull grey);
   - **kill boring scenes** — any scene that stalls or has "not much going on" gets a 2nd/3rd act + payoff beat; must ESCALATE across its full duration;
   - **add themed per-scene SFX** + transition slams;
   - **add risers + retention hits wherever possible** (risers on builds, pattern interrupts like flung sprites/bursts/confetti, countdown, progress-bar reward);
   - **on-screen text that is NOT a repeat of the VO** (punchy labels/callouts that add a second info layer);
   - **costumes** on every sprite, consistent across the reel.
   Exit test per scene: interesting + escalating + has SFX + not boring + text != VO. Keep looping (typically 2-3+ passes).
5. **⛔ FINAL adversarial critic pass with a FRESH EYE.** I'm too lenient on my own work, so **spawn an adversarial ship-gate critic subagent**: give it the contact-sheet PNG (it Reads the image) + the checklist below + the reel's intent, tell it to list EVERY fail harshly (neon/dull-grey, thin/boring/non-escalating scene, missing SFX/risers, bare-uncostumed sprite, small/occluded hero, cramped, unclear reference, on-screen text == VO, weak hook, desync, em dashes). FIX every fail → re-render → re-run until it passes.
6. **Deliver** the finished reel to Final + Drive + a ONE-LINE note. No checklist dump.
7. **Any revision from Alex → fix → RE-RUN the loops + gate → re-deliver.** Never hand back a fix without re-gating the whole reel.

## THE SHIP-GATE CHECKLIST (fix any fail before delivering)
- **Style:** warm cel-shaded ANIMATION palette; NO neon-on-black; NO dull grey (tint every neutral fill); looks hand-animated, not vibecoded; characters are the **costumed Claude pixel sprites** (clay Mascot + scene-appropriate outfits — hats/suits/helmets/masks/armor/capes), consistent across every scene, never bare/placeholder. (Ref: [[claude-ai-reel-workflow]] premium-polish; CREW sprite+costume system + the neon lesson in [[crew-reel]].)
- **⛔ CONTAINED FRAME (Alex 2026-07-08, MINT — HARD rule, he was furious):** ALL scene content lives INSIDE the dark rounded `Panel` (the "black square", left:34 right:34 top:384 height:792), exactly like reels 31-34 (BLUEPRINT/AUTOPILOT/CREW). The scene HEADER goes INSIDE the Panel (top ~74), NOT at the very top of the frame (IG chrome hides it — Alex: "remove that completely"). Captions render BELOW the Panel (~y1256). NEVER float elements loose on the cream background (no browser/sprite/streams/ghost-boxes floating outside the frame), NEVER a mascot half-out-of-frame, NEVER a top-of-frame text line. The progress bar is at the very top; the Panel sits below it. If a scene isn't wrapped in `<Panel>`, it's WRONG. (The v1-v3 MINT floated everything on cream and looked broken — the single biggest failure of the whole build.)
- **Hero & layout:** hero big + centered + real depth (bloom/contact-shadow/stacked navy shadow); NOTHING critical occluded (timer/logo/text over a face, moving object over a figure); nothing cramped/too-close; IG safe zones (top ~250 / bottom ~340 / right ~120). (Ref: [[reel-ig-feed-safezone]], [[reel-scene-motion-depth]].)
- **Scene richness + motion:** 15-30+ animated elements; ESCALATES across the FULL scene (no climb-then-freeze; build a 2nd/3rd act); ends on a payoff beat; ZERO dead/boring stretches; z-layers/parallax/secondary motion. (Ref: [[reel-scene-motion-depth]].)
- **Reference clarity:** any movie/pop-culture reference unmistakable within 1s AND detailed; on-screen scene text NEVER repeats the VO. (Ref: [[crew-reel]].)
- **⛔⛔ SPRITE-DRIVEN, NOT UI SCREENS (Alex 2026-07-08, MINT — the biggest lesson of the whole build).** Reels must be **sprite-driven cartoon SCENES where the Claude Mascot sprites ACT OUT the concept** (like CREW's movie scenes, or CLONE's rocket), in **warm colorful animation backdrops** (a different color per scene). Do NOT build literal UI/browser/app screen mockups, even for a "browser/tech" idea, they read BORING and "practically everything is wrong" (Alex's words). Represent the info **metaphorically through sprites**: e.g. browser-automation = a hero sprite grabbing a giant cursor and blasting a mountain of busywork while a crowd watches; "hundreds of jobs" = a sprite firing paper-airplane applications at buildings; "post everywhere" = a sprite with a megaphone to platform-sprites; "it learns by watching" = two sprites (one does it, one records with a notepad). A dark uniform screen / neon-on-black / lines-and-boxes is WRONG. The HOOK especially must be a **dramatic scroll-stopping sprite scene** (crowd reacting to something big), not "just a screen doing something." Reference: the MINT sprite rebuild that finally worked, and [[crew-reel]]. (Ref [[mint-reel]].)
- **Retention:** mute-readable hook-header ([[reel-hook-header]]); big countdown / open loop; a pattern interrupt or two; REHOOK visually differs from the hook; top progress bar + reward seal that unlocks ([[reel-progress-bar-reward]]).
- **Audio/SFX:** THEMED per-scene SFX (not just generic entrance slams); risers on build moments; satisfying slam/impact on each scene transition; music ducked under VO. (Ref: [[crew-reel]] themed-SFX layer, [[video-editing-toolchain]].)
- **Captions:** WAV-onset anchored + drift-gated, synced, no desync; fragments merged. (Ref: [[caption-sync-gate]].)
- **Script/structure:** subject = the hot model (Fable 5); deadline + loss in line one; GATE THE PAYOFF (every beat a RESULT, never the how); comment CTA at the END + in caption (never open with it); specific NAMED personas; best-first order; value in the first breath; hard cut on the keyword. (Ref: [[reel-winning-formula]], [[no-anecdote-value-first-scripts]].)
- **Hygiene:** ZERO em/en dashes anywhere (incl. captions/guide) ([[no-em-dashes-in-copy]]); delivered to Final + Drive ([[social-assets-to-gdrive]]); correct 1080x1920 / 30fps / duration.
