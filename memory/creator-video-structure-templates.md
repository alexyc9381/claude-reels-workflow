**GREG ISENBERG — "numbered-promise" explainer/listicle (his long-form/podcast-cutdown format):**
1. HOOK lives in the TITLE = **[specific number] + [bold $/value claim] + [low-barrier promise]**. e.g. "7 $1M+ AI startup ideas you can launch tomorrow with $0" (number=7, claim=$1M+, barrier=tomorrow/$0). Recurs across his catalog.
2. COLD OPEN restates the quantified promise out loud, verbatim ("I promise you you're gonna have like seven AI apps you can build…"). Tell them exactly how much value is coming before any content.
3. BODY = explicit NUMBERED LIST — discrete labeled chapters ("Idea 1: …", "Idea 2: …"), each preceded by a short intro beat.
(Per-item internal structure, pacing, ending/CTA = NOT confirmed.)

**NATE HERK (Nate Herkelman, n8n/AI-agent creator, founder Uppit AI) — build/tutorial template:**
1. HOOK = TITLE promises a complete, impressive **end-state build + no-code/accessibility angle** ("I Built the Ultimate Team of AI Agents in n8n With No Code"). Finished ambitious system + "you can build this even with no code."
2. BODY ARC = **problem → solution → live build**: state the problem (e.g. "one agent gets overwhelmed"), name what's being built (a centralized orchestrator coordinating specialized sub-agents), then implement it live, no-code.
3. SEQUENCING = **progressive complexity**, chunked ordered modules: basics → first workflow → AI agents → RAG/vector → APIs → error-handling/best-practices.
4. SIGNATURE = give the viewer a **mental-model architecture** (orchestrator + specialized sub-agents, "so no single agent gets overwhelmed") that organizes the whole video.
5. CTA = off-platform — drives to a **free template + Skool community**.

**⭐ SKILL-STACK LISTICLE FORMAT (from a viral IG reel, DZSFkNppVW_, transcribed 2026-06-23 — Alex wants scripts like this).** Current high-performing format: (1) HOOK "Did you know you can now run your entire [X team/function] inside Claude?" (the carousel-creator "Did you know Claude can now…" formula scaled to a whole TEAM); (2) "So before you spend [thousands of dollars and hundreds of hours] on [X], try this."; (3) a NUMBERED list of FIVE specific, named SLASH-COMMAND skills, each framed as a contrarian replacement: "Number N, instead of [expensive/painful old way], try /[command]. [one line on what it does + a hard number]." Real examples from the source: "/spy pulls every competitor's active ads with hooks, CTAs, offers, and how long each has run", "/bulk creative churns out 20 on-brand copy variations in 10 minutes", "/ad score grades hook, copy, CTA, emotional resonance, offer, visual fit before you launch", "/ads meta runs 186 checks on creative fatigue + audience overlap for an ads-health score 0-100"; (4) social proof "hundreds of agencies are already running the stack"; (5) gated CTA "Comment KEYWORD if you want the full install guide for all five." Why it works: it NAMES the skills (concrete + NEW = Claude Skills / slash commands), "instead of [old way]" frames each as killing an expensive tool/role, and the install guide is the gated payoff. The named commands are the WHAT (shown on screen); the build/install is the HOW (gated) — honesty: the gated doc must actually deliver buildable skills. Runs ~60-70s for 5 items (longer than the 130-150w single-capability scripts — inherent to a listicle); cut to 4 items for ~50s.
