Alex likes IG @1609plus's post style and wants it replicated for Matchtern. Working replica pipeline lives at `~/Downloads/matchtern-1609style/` (`build.js` = Node + Playwright, renders 9 PNGs to `out/` at 1080x1080 @2x; `caption.txt`).

**@1609plus visual system (their flagship is a 9-slide product-feature tour):**
- Near-black canvas `#060709` with a faint blue radial glow + vignette + subtle SVG film grain overlay.
- Cover: giant white wordmark (Inter ExtraBold, letter-spacing ~ -5px) + one-line gray subtitle + a glowing 3D-tilted brand object (electric blue, heavy bloom, floor reflection).
- Content slides: tiny brand mark top-left, `NN / 09` counter top-right, white headline ExtraBold ending in a period, gray subhead, then ONE dark glassmorphic UI-mockup card (`rgba(18,20,27,.72)`, 1px `rgba(255,255,255,.08)` border, radius ~22, blue-tinted shadow). "Swipe →" bottom-right; last slide = `@matchtern`.
- UI cards mimic a real product: conic-gradient score ring, reach/match/safety list with colored left borders, roadmap node graph with glowing connectors, leaderboard with highlighted "You" row, AI before/after rewrite card, mentor card, pricing card.
- Accent = electric blue. For Matchtern use logo-blue `#2440BD` + brighter `#3E63FF` for glow (nearly identical to their blue, so it's both faithful AND on-brand). Secondary: check-green `#1F9E47`, stripe-blue `#5D8CC4`, cross-red `#D32F2F`. See [[matchtern-design-system]].
- This is the DARK style, the opposite of the light editorial [[anthropic-skills:matchtern-carousel]] format. Use this only when Alex explicitly asks for the 1609plus look.

**IMPORTANT (learned the hard way):** "1609 style" = the VISUAL skin (dark/glowy/glassmorphic), full stop. When Alex asks for a "value post" or "matchtern style" post that "doesn't pitch what Matchtern is," he is changing the CONTENT (educational/value-first, soft brand, no pricing or product UI), NOT the visual style. Keep the dark 1609 visuals on EVERY post unless he explicitly asks for the light editorial look. Do NOT switch to the light Playfair/off-white system just because a post is value-first. A value post lives at `build_value.js` (7 slides, dark 1609 skin, "land an internship with zero experience" method) -> `out_value/`.

**Legibility (Alex's standing note):** 1609's authentic UI text is small, but Alex wants the BODY graphics and in-card text on content slides sized for phone viewing. Floor sizes on the 1080px canvas: in-card body text >= 22px, list/row items 22-23px, card titles 26-32px, data-viz (radar/charts) large enough to dominate the card with axis labels >= 17px. Subheads 24-25px. When in doubt, bigger. The first spike-post draft was rejected for being too small; the enlarged pass is the reference baseline.

**Shared engine (use this for all NEW posts):** `engine.js` exports the whole dark 1609 system at the enlarged/legible baseline — `frame, head, cover, dot, statBars, compare, numberedCard, iconCards, listRows, emailCard, radarSVG, glowTile, followChip, centeredClose, callout, commentUnlock, render, C(palette)`. A build script is just `const E=require('./engine.js')`, push `[name, slide]` tuples, `E.render(slides, outDir)`. Read engine.js's doc header for signatures. Batch-build pattern that works well: fan out one subagent per post, each given a slide-by-slide spec referencing these helpers, each self-QAs its own previews.

**Carousels built (each its own build_*.js -> out_*/ + caption_*.txt):** `build.js`/`out/` (product pitch), `build_value.js`/`out_value/` (zero-experience method), `build_spike.js`/`out_spike/` (spike>well-rounded), then the 5-post value batch on the shared engine: `build_list.js`/`out_list/` (7 startup jobs for a 16yo), `build_wrong.js`/`out_wrong/` (wrong/prestige internships), `build_emails.js`/`out_emails/` (5-email sequence + comment-INTERN unlock), `build_ways.js`/`out_ways/` (5 ways to land one — the ONE post that names Matchtern, soft, on slide 5 + matchtern.org CTA), `build_proof.js`/`out_proof/` (internship vs 1500 SAT).
