Most people don't realize that they can set up 4A agents to work together as the dev team to ship out features while you are asleep. This is how you set it up. Create 4 specialized agents. The planner agent takes a vague feature request and turns it into a detailed spec with the exact file pass, functions, signatures, and edge cases. Use Opus 4Play as the driver because the quality of a pan sets the ceiling for everything else. The coder agent reads the spec and builds exactly what it says. Use Sonic 4.6 to save costs because it has already an exact implementation path. The tester agent reads what the coder builds and writes test cases covering the edge cases and the happy path. The viewer agent is read only. It cannot edit any code. It reads the spec, the changes, runs it get diff, and gives a verdict before touching the main branch. This is the most important part. Each agent writes to a shared folder called .pipeline where the next agent can read it. For example, the planner agent gets his from .pipeline.md and the coder agent gets his from .pipeline.md and so on. This is necessary so that every agent has contacts from the agent before it. Create one slash band that changes all of them together so that when you have a feature like Agray limiting to the login endpoint, it hands directly to the planner agent through to the reviewer agent at the end of the loop.
