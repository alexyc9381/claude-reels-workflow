# nateherk — Script DNA
> Source: 11 transcripts (YouTube auto-captions, cleaned), 61K–161K views, fetched 2026-07-12. 7 are shorts (~420–670 words, Alex's target format); 4 are 6–10min long-forms mined for hook/proof/CTA craft. Highest-view shorts weighted most: Google GWS (161K), $10k Apple sites (148K), 99% cheaper (105K).

## 0. Voiceprint
Fast, confident builder-to-builder show-and-tell that opens on a finished result, narrates the screen in real time with reaction beats ("boom," "Wow, I mean…"), stacks real numbers and named companies as proof, and gates the full method behind a link — casual, "you guys," slightly contrarian, never a beginner-explainer.

## 1. Opener / Hook Formula

### Primary template — the "Here's how" result-first hook  (4 of 11; 3 of top-7 shorts)
```
Here's how [I build / you can] [DESIRABLE OUTCOME] [with Claude Code / instantly / for free].
```
The value noun lands in the first 6–10 words, the outcome is outcome-framed (money, speed, free), and the very next sentence starts the demo. No preamble, no "in this video."

Verbatim:
- "Here's how I build 3D animated websites with Claude code." — $10k Apple sites (148K)
- "Here's how you can use Claude Code for completely free." — 99% cheaper (105K)
- "Here's how I build end workflows instantly with cloud code." — n8n (65K)
- "Here are five easy hacks to build beautiful websites with Claude Code." — 5 hacks (67K)

### Variant A — news-drop / "it just dropped"  (2 of 11, both news-reactive)
```
[Company] just dropped [what some are calling the most powerful X]. / Well, [Product] is finally here. Or is it?
```
- "Google's new tool lets you connect to anything instantly. Google just dropped what some are already calling the most powerful workspace CLI on the internet." — Google GWS (161K, his single biggest)
- "Well, Claude Mythos is finally here. Or is it really?" — Mythos (100K)

### Variant B — cold-open result reveal ("what you're watching is not what you think")  (2 of 11)
```
[Point at the finished artifact on screen]. What you're watching / looking at right here is [X]. This was [not filmed / not built the way you assume].
```
- "So, I literally just opened up Claude Fable, gave it this /goal, went down to the gym, and came back to this. What you're watching right now was not filmed. This avatar is AI." — Fable 5 (146K)
- "What you're looking at right here is 36 of my most recent YouTube videos organized into an actual knowledge system that makes sense." — Karpathy (64K)

### Variant C — bold-claim / equation hook  (2 of 11)
```
[Big claim as fact] / [Tool] + [Tool] = [$ outcome].
```
- "Right now, a brand new group of millionaires is being created by AI." — Millionaires (116K)
- "Seedance 2 plus Cloudcode builds $10,000 websites." — Seedance (61K)

### The 5 rules that make the hook fire
1. **State the result, not the topic.** He never says "today I'll talk about workflows"; he says "Here's how I build workflows instantly." The finished thing exists in sentence one.
2. **Outcome-frame in the hook itself** — money ("$10,000," "millionaires"), speed ("instantly," "in 5 minutes"), or cost ("completely free," "99% cheaper").
3. **Value noun by word ~7.** "websites," "workflows," "free," "millionaires" all arrive fast.
4. **The second sentence is already the demo or the twist** ("This is one that I built with Claude code…this website is not Apple").
5. **News hooks lead with the source's own hype, then undercut it** ("what some are calling the most powerful…"; "finally here. Or is it really?").

## 2. Structural Skeleton (the beat map — SHORT-reel spine, Alex's format)

Nate has no named-section grammar (no "The Scanner." like raycfu). His structure is a **screen-recording walk-through wrapped in a result → CTA sandwich.** Ordered beats:

1. **RESULT HOOK** — the finished outcome, stated as fact (see §1). *Fn: stop the scroll with an outcome.*
   - "Seedance 2 plus Cloudcode builds $10,000 websites."
2. **THE REVEAL / "not what you assume"** — the surprise that makes it worth watching. *Fn: create the gap.*
   - "this website is not Apple. This is one that I built with Claude code using this new skill in like 30 minutes."
3. **THE ENABLER NAME-DROP** — the one tool/skill/file that makes it possible. *Fn: give the video its keyword.*
   - "using this new skill"; "the front-end design skill"; "this one skill…is called grill me."
4. **NARRATED WALK-THROUGH** — screen narration in present tense, partial not complete (gate the how). *Fn: prove it's real, withhold the full recipe.*
   - "And now what it's doing is it's creating the Google Doc. Not via API call. Not via MCP. But via a bash command."
5. **REACTION PAYOFF** — a live "it worked" beat. *Fn: dopamine + credibility.*
   - "And we can see boom. We have an actual resource guide."
6. **ONE-LINE ZOOM-OUT** — why it matters / who it's for. *Fn: breadth + stakes.*
   - "But there's so many different benefits here using this workspace CLI."
7. **CTA** — gate the depth (see §8). *Fn: send to the full breakdown / the DM.*
   - "If you want to watch the full breakdown, then click on that play button right here."

**Long-form spine (mined, not Alex's length):** hook → context primer ("in case you're not super deep…let me give you some context") → the four/five reasons or steps → third-person receipt (Vulcan, YC) → the 4-step "how you start today" → community gate → like/outro.

**Target length / budget:** Alex reel = 35–45s ≈ 150–190 words. Nate's shorts run 420–670 words at faster delivery; compress to his 6-beat spine and one reaction beat.

## 3. Per-Beat Micro-Templates

- **Result hook:** `Here's how to [OUTCOME] [with Claude / instantly / for free].` or `[Tool] + [Tool] = [$OUTCOME].`
- **Reveal:** `This isn't [the obvious assumption]. This is [what it actually is], built in [surprisingly small time].`
- **Enabler name-drop:** `And all it takes is [one skill / one file / one tool] called [KEYWORD].`
- **Walk-through (gated):** `So [watch what it does]: it [action], then [action], then [action]. Not [the slow old way]. [The new way].`  ← show the WHAT, never the full prompt.
- **Reaction payoff:** `And boom — [the finished artifact]. [One concrete detail that proves quality].`
- **Zoom-out:** `And that's just one example. [Anyone from X to Y] can do this.`
- **CTA (Alex version):** `Comment [KEYWORD] and it's yours.`

## 4. Sentence Rhythm & Mechanics

- **Opener length:** avg ~9 words (range 6–12). "Well, Claude Mythos is finally here." (6) → "Here are five easy hacks to build beautiful websites with Claude Code." (12).
- **Avg sentence length:** ~16–20 words, but strongly **bimodal** — long run-ons stitched with "and/so" against hard fragments. Fragments are the signature: "Boom." "Nice." "Ooh, that's super nice." "Not via API call. Not via MCP." "Mythos-class."
- **Imperative vs declarative:** hooks and how-to steps skew imperative ("go to ollama.com," "click on models," "pick one real task," "make it argue with you"); demo narration is present-tense declarative ("now what it's doing is it's creating the Google Doc"). Roughly 25–30% imperative in teaching stretches, near-zero during pure demo narration.
- **Number / $ density: very high.** Named figures land every few sentences — "$10,000," "$65 billion," "$965 billion," "$47 billion," "47 times growth," "20 bucks a month," "$200 a month plan," "10 bucks per million input tokens, 50 on the output," "1100 nodes," "99% property accuracy," "2,700 workflow templates," "in 5 minutes," "in like 30 minutes," "60 or 70% right," "95% of people." Aim for a number or dollar figure in the hook AND the payoff.
- **Causal connectors:** heavy **"So,"** sentence-starts (his single most frequent opener word — "So, here you can see," "So, it can search," "So, let me show you guys"), plus "And," "But," "Now," "Because." "So" = the connective glue of his cadence.
- **Signature cadence device:** **anaphora-then-reveal** — repeat a frame, then flip it. "Not via API call. Not via MCP. But via a bash command." / "I didn't write this, I didn't film it, I didn't edit it."
- **Punctuation:** comma-spliced run-ons and mid-sentence self-interrupts ("I mean," "you know," "obviously," "literally"). No em-dashes needed — his cadence is already comma/period-driven, which fits Alex's rule natively.

## 6. Retention / Escalation Devices

1. **Open loop / forward promise:** "So, let me show you guys what this thing is actually good at, and then exactly how it made this video." (Fable 5) — names two payoffs to keep you watching.
2. **Live reaction beats** re-hook mid-demo: "boom," "Wow," "Nice," "There we go, right on cue" — proof the thing is running unscripted.
3. **The anaphora reveal** as a mini-cliffhanger: "Not via API call. Not via MCP. But via a bash command."
4. **Contrarian re-hook:** flip the expected method. "you shouldn't prompt your agents anymore. You should have loops of agents that prompt your agents." / "a skill doesn't have to be super complicated…A skill can just be a prompt."
5. **Number-drop escalation** in long-forms: "$1 billion → $47 billion, 47 times growth" — stacks figures to build awe.
6. **"Here's the trick":** "And the trick is you can't just generate like four straight minutes of audio because…the voice starts to drift." — insider tell that rewards the viewer.
7. **Self-aware honesty as a hook:** "I haven't watched this yet, so hopefully it's good." / "that's my horrible visual" — lowers guard, feels live.

## 7. Proof & Credibility Style

- **Third-person receipts (Alex-safe, his strongest proof):** Vulcan ("won a Virginia state contract…at around 10% of the price," "saving Virginians over a billion dollars a year," "most of their team can't even write a single line of code"); Y Combinator ("over half of the startups are building with Claude…the most used AI in the entire batch"); Stripe ("Fable 5 compressed months of engineering into days").
- **Company/people name-drops as authority:** Anthropic, OpenAI, Airbnb/Stripe/DoorDash, Tanner Jones, Matt Pocock, Andrej Karpathy, Y Combinator; tools: Nano Banana, Cling, Seedance, 11 Labs, HeyGen, Ollama, Qwen, n8n.
- **Hard numbers over adjectives:** funding ($65B raise, $965B valuation), run-rate ($1B → $47B), pricing ($10/$50 per million tokens, "20 bucks a month," "$200 plan"), accuracy ("99% property accuracy," "1100 nodes"), success rates ("95% vs 80%").
- **Show the artifact on screen** as the ultimate proof — the resource guide, the live localhost site, the Obsidian graph, the finished n8n workflow. The receipt is visual, the VO points at it.
- **Honest hedges keep it credible:** "which I always think you should take with a grain of salt," "reportedly," "they claim," "I could be completely wrong." He never overclaims a benchmark.

## 8. CTA Formula

**Nate's two native CTAs:**
- **Shorts (7 of 7):** "If you want to watch the full breakdown, then click on that play button right here, and I'll see you guys over there." — gates DEPTH to the long-form.
- **Long-forms:** community gate — "if you guys want a shortcut for all of this, I put everything in my free school community. The link for that is down in the description…there's a 7-day challenge…The [row/grill me] skill…that's in there as well for completely free." Then like/outro tail: "please give it a like…I appreciate you guys making it to the end…I'll see you on the next one. Thanks, guys."

**Keyword mechanics:** the video's *enabler name* IS the gate — "grill me," "the front-end design skill," "GWS." The full skill/file/prompt is never spoken in full; it lives in the community/description.

**Alex adaptation:** keep the enabler-name-as-keyword; drop the "play button" and the "see you over there / thanks guys" tail. Cut hard on the keyword: `Comment [KEYWORD] and it's yours.`

## 9. Why It Works (transferable mechanism)

- **Result-first kills the scroll.** The finished outcome exists in sentence one, so the viewer's only question is "how," and that question carries them through the demo.
- **Show-don't-claim.** The artifact is on screen; the numbers are real and named; hedges keep it honest. Trust compounds beat by beat.
- **Gated depth = engineered curiosity.** He shows the WHAT (it works, here's the result) and withholds the HOW (the full prompt/file), which is exactly what powers the CTA.
- **Reaction beats make it feel live,** not scripted — "boom," "hopefully it's good," "there we go" read as a real person discovering it with you.
- **Outcome + number framing travels** — "$10k website," "millionaires," "99% cheaper," "in 5 minutes" are legible to anyone, not just engineers.

## 10. Adapting to Alex (the fusion layer)

**Transfers cleanly:**
- Result-first hook, outcome/number framing, the enabler name-drop, present-tense screen narration, reaction beats ("boom"), the anaphora reveal, third-person receipts, gated-how structure, high number density, comma-driven cadence (already em-dash-free).

**Conflicts with Alex's HARD RULES → resolutions:**
- **First-person anecdote (his default mode) vs Rule 2.** Nate is heavily "I built / I came back to this / How I build." → **Rewrite to second-person or agentless:** "Here's how I build 3D websites" → "Here's how to build 3D websites"; "I literally opened Claude Fable and came back to this" → drop the gym story, keep the capability + a third-person receipt. **Keep** his third-person receipts verbatim in spirit (Vulcan, Stripe, YC) — those are Rule-2-safe and are his best proof.
- **Gate the how (Rule 3):** already native — he shows result + partial narration, gates the full prompt/file. Keep the WHAT, never voice the full method.
- **Cut on keyword (Rule 4):** trim his tails. Remove "click on that play button right here, and I'll see you guys over there" and "thanks, guys." End on the keyword line.
- **No em-dash (Rule 1):** no change needed; his cadence is comma/period-based.
- **12-year-old parse (Rule 5) + breadth litmus (Rule 7):** favor his SHORT-reel vocabulary (websites, workflows, free, inbox), not his long-form insider terms (hyperframes, GSAP, sub-agent orchestration, context rot). His reels are TOFU; keep them TOFU.
- **Never fabricate (Rule 6):** he name-drops real engineers — keep authority beats true and general, never invent a quote for a named person.

**Gate-the-how mapping for this format:** SHOW = the finished artifact on screen, the enabler's *name*, the 3 steps in plain verbs, one real number. GATE = the full prompt, the skill file, the exact config — behind the comment keyword.
