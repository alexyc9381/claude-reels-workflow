## The Creative Studio Skill Stack — from "7 Claude skills to run an entire creative studio" (292,400 views)
- **What it does:** turns Claude into a one-person content agency: it animates video, spies on competitor ads, runs market research, writes in your exact voice, turns docs into podcasts, and ships fact-checked copy, all from one workspace.
- **The build** — this is not one system, it is a *bundle* of 6 skills she frames as 7 (see the honesty flag). Each is a Claude Skill / project-instruction / connector:
  1. **Remotion motion skill** — a Claude Code Skill wrapping a Remotion (React video) project. You describe a shot; the skill writes/edits `.tsx` compositions and renders. This is literally Alex's own `claude-reels-workflow` chassis, so he already owns it. *Prompt shape:* "here is the scene / VO / text card, generate the Remotion composition and render at 9:16." The trick: a pre-scaffolded Remotion repo so Claude only edits components, never bootstraps from zero.
  2. **Competitor Ads Extractor** — pulls live competitor ads (messaging, hooks, CTAs). Claude cannot do this natively; it needs a data source: the **Meta Ad Library API** or an **Apify actor** wired in as an MCP/connector. *Prompt shape:* "pull the last N active ads for [brand], cluster by hook, summarize the angle + CTA."
  3. **Deep Research agent** — the multi-step background researcher. Real: this is the `deep-research` skill / Claude's research mode. *Prompt shape:* a research question + "run market analysis, cite sources, return a brief."
  4. **Voice DNA** — extracts your sentence rhythm + vocabulary so future copy sounds like you. This is exactly what Alex's `script-style-replicator` already does. *Prompt shape:* "ingest these 5-10 samples, extract a voiceprint (cadence, diction, openers), save as reusable style instructions."
  5. **ElevenLabs (she says "11 laps")** — doc → narrated audio or 2-voice podcast in ~1 min. Real via the **ElevenLabs MCP** (paid; she admits this is the one that "needs a subscription"). *Prompt shape:* "turn this doc into a 2-host podcast script + TTS render."
  6. **Content Research Writer** — research + copywriting + citation fact-check in one pass. Buildable as a single Skill that chains web search → draft → verify. *Prompt shape:* "research [topic], write [format], fact-check every claim with a linked citation."
- **The wow artifact:** the *count itself* on screen. Seven skill cards / seven install lines flashing past, each doing a different agency job, so the viewer feels an entire company assembling inside one Claude window in under 40s. The single hardest-hitting frame is Remotion turning a text prompt into a moving explainer clip with "no editing."
- **⚠️ Buildable today? PARTLY.** Four of the six are genuinely real right now (Deep Research, Voice DNA, Content Research Writer, Remotion-with-scaffold). Two are oversold: **Competitor Ads Extractor** is smoke as a "skill" — Claude can't pull live ads without a Meta Ad Library / Apify connector doing the actual scraping, so the guide must ship that connector or it fails on step one. **ElevenLabs** works but is paid, not "free." And the video says "seven skills" but only names six (the transcript lists 6) — do not promise seven or the guide is caught lying. Honest framing: "6 real skills, 4 free, 2 need a connector or a paid key."
- **The gated deliverable:** the actual install pack. (a) The Remotion skill repo + the one-line install; (b) the exact Meta-Ad-Library/Apify connector setup with the config JSON; (c) the ElevenLabs MCP connect steps; (d) the copy-paste Voice DNA + Content Research Writer + Deep Research skill prompts. ## The Unrejectable Resume Loop — from "The 4 Skills that make your resume unrejectable" (25,600 views)
- **What it does:** runs your resume through a 4-stage pipeline (diagnose, keyword-match, rewrite, interview-prep) so it survives the ATS filter and you walk into the interview already rehearsed.
- **The build** — a 4-skill pack, each a project instruction or slash-command; no external tools required beyond Claude + optionally web search:
  1. **The Diagnosis** — emulates an Applicant Tracking System: parses the resume, flags parse-breaking formatting, missing sections, weak bullets. *Prompt shape:* "act as an ATS parser; score parse-ability, list every section that would drop out, and why."
  2. **The Recruiter** — compares your resume to the norms of your target role and returns the missing keywords/skills. *Prompt shape:* "for a [role] in [field], list the keywords/skills top candidates show that this resume is missing." (She says "against 1,000 job descriptions" — see the flag; realistically it uses trained field knowledge unless you paste or web-search real JDs.)
  3. **The Rewriter** — rewrites every bullet on the **Google XYZ formula** ("Accomplished X as measured by Y by doing Z"). Pure prompt. *Prompt shape:* "rewrite each experience bullet as XYZ, quantify Y wherever possible, keep it one line."
  4. **The Hiring Manager** — turns Claude into the interviewer: asks the hardest role-specific questions, then scores your answers out of 10 with fixes. *Prompt shape:* "you are the hiring manager for [role]; run a live interview, one hard question at a time, then grade each answer /10 with a stronger version."
  The trick: the four run as a *loop* (diagnose → score → rewrite → prep), and the persona-naming ("the diagnosis / the recruiter / the rewriter / the hiring manager") is what makes four prompts feel like a system.
- **The wow artifact:** the ATS "diagnosis" screen lighting up red flags on a resume in <2s (the "your resume is sick" reveal), and the closing "/10" score on a mock interview answer. Both are instant, gamified state-changes.
- **⚠️ Buildable today? BUILDABLE (with two honest approximations).** The rewriter and the hiring-manager interview are 100% real, pure-prompt, works today. The two soft spots: "scans like a **real** ATS" — Claude emulates ATS heuristics, it is not literally Workday/Greenhouse's parser (close enough to be genuinely useful, but say "ATS-style check," not "the real ATS"). And "analyzes against **1,000** job descriptions" is smoke — Claude isn't reading 1,000 live postings; it's using field knowledge. To make that real the guide should tell users to paste 5-10 real target JDs (or enable web search), then it honestly delivers the keyword gap.
- **The gated deliverable:** the four copy-paste prompts as a saved project (Diagnosis / Recruiter / Rewriter / Hiring Manager), plus the "paste 10 real job descriptions here" step that makes the recruiter stage truthful, plus the XYZ bullet template and a scoring rubric for the mock interview.
- **Alex's angle:** hook = *"Did you know Claude can now make your [cold email / pitch deck / LinkedIn profile] basically un-ignorable?"* Use cindiezhu's own 25.6K teardown formula (coined power-word + "most people just paste X and say make it better, that's why it's not working" call-out + 4 personas + recap quad). The resume topic itself is already huge-TAM; if Alex wants a fresh angle the teardown's "un-ignorable cold email / Comment EMAIL" variant is broader still. Keep the persona system and the "/10" score, add the gate.

## The One-Photo Room Redesigner — from "Claude x Nanobanana can redesign your entire house now" (12,300 views)
- **What it does:** upload one photo of your room, type a request, and get back photorealistic redesigns (new furniture, colors, layouts) you can iterate on endlessly, replacing a $3,000 furniture gamble or a hired interior designer.
- **The build** — a Claude Code Skill that orchestrates Google's image model:
  1. Install a "banana" **Claude Code Skill** (she says: Google "banana skill claude," open the first result, copy the URL, `install this skill` inside Claude Code). The skill is the wrapper.
  2. Connect **Google's Nano Banana image model** (Gemini 2.5 Flash Image) via a **Google AI Studio API key** — create the key, paste it back into Claude, `set this up`. This is the one real friction point and the skill stores the key.
  3. Payoff: `/banana` + upload room photo + "redesign this room like a professional interior designer." Claude passes the image + instruction to the Gemini image model, which renders the redesign; then you conversationally iterate ("change the couch, warmer palette, open layout").
  *Division of labor:* Claude directs and manages the loop; **Gemini/Nano Banana does the actual rendering.** The skill is glue.
- **The wow artifact:** the before→after room swap. One frame the room is the user's real cluttered space; <2s later it is a magazine-grade redesign of the *same* room, same windows, new everything. That identity-preserving edit is the whole reason it broke out.
- **⚠️ Buildable today? BUILDABLE (one honest correction).** Nano Banana (Gemini image) is genuinely strong at identity-preserving room edits, so "actually looks good" is real, not hype. Real caveats to state: it needs **Claude Code** (not the chat app), a **Google AI Studio API key** (free tier is limited; heavy use is paid), and "Claude does everything" is slightly overstated because **Gemini renders the image** — Claude orchestrates. None of that is smoke; it just needs the setup the reel compresses into 20 seconds.

## Carousels as Visual Code — from "How people are creating these 10/10 carousels" (7,564 views)
- **What it does:** generate a full, on-brand Instagram carousel (every slide titled, copywritten, consistently branded) from one topic line, with no design tool, then export to Canva if you want to hand-edit.
- **The build** — pure native Claude, no external tools:
  1. Create a Claude **Project** named "Instagram carousels."
  2. Paste a **system prompt** into the project that tells Claude to build each carousel as **visual code** (an HTML/React **Artifact**) rather than text — code is what enforces consistent layout, fonts, and color across slides. (This prompt is the gated asset.)
  3. Type a topic into the chat; Claude generates the whole deck as a **clickable interactive Artifact preview** you can page through.
  4. Edit conversationally ("fix slide 3," "tighten the hook") or upload brand guidelines; Claude can also *generate* brand guidelines if you have none.
  5. Escape hatch: export the Artifact to **PDF → import into Canva** to make it a hand-editable file.
  The trick: "carousels as visual code" — by having Claude emit a rendered HTML template instead of prose, every slide inherits the same grid and brand tokens, which is why the output looks designed instead of like pasted text.
- **The wow artifact:** the interactive Artifact appearing and the user *swiping through* a finished, branded 8-slide carousel that was one sentence a moment ago. The "you can click through it" tactility (it's real, not a mockup) is the payoff frame.
- **⚠️ Buildable today? BUILDABLE.** This is the most honest of the four — Claude Artifacts render HTML/React carousels natively, projects persist the system prompt, and the whole loop is real with zero paid dependencies. Two small truth-checks: the Canva "PDF → editable file" import makes *elements* editable but is imperfect (text may import as blocks, not always live text layers), and "consistent branding" is only as good as the brand tokens in the prompt. Otherwise legit.
- **The gated deliverable:** the actual "build carousels as visual code" system prompt (the whole thing she references but never reads aloud), a brand-guidelines generator prompt, the slide-template structure, and the Canva-export steps. Note: this video is the one top-4 hit with **no comment gate** (it ends on a benefit line), and the teardown flags that as why it under-performed her others. Alex must ADD the gate his version.
